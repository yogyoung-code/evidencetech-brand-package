## 1. 平台定位矩阵

每个平台先回答三个问题：**面向谁、目标是什么、不做什么**。这张表是社媒决策的第一个 checkpoint——任何选题进入排期前先对这张表。

| 平台 | 主受众 | 我们的目标 | 内容主轴 | 节奏 | 明确不做 |
|---|---|---|---|---|---|
| 微信公众号 | 临床医生 / 药企商务 / 投资者 | 树立专业权威 + 留资转化 | 长文：循证方法论、产品案例、行业评论、Demo 引流 | 1–2 篇/周 | 标题党、震惊体、互推号召 |
| LinkedIn（CN） | 药企商务/医学/KA、行业高层 | 行业领导力 + 招聘 + BD 触达 | 短帖+图：方法论、活动、招聘、合作公告 | 2–3 篇/周 | 私域催单、高频转发自家公众号 |
| LinkedIn（EN） | 海外药企 / 学术圈 / 投资者 | 海外身份锚定（被动存在感） | 月度精选中英对照、季报 highlights | 1 篇/周 | 大量翻译再发、海外获客（独立立项） |
| 微博 | 行业从业者、媒体记者 | 实时反应 + 媒体联动 | 短帖+卡片：观点、活动播报、引述权威 | 按事件 | 蹭社会热点、医患话题、明星议题 |
| 知乎 | 医学生 / 年轻医生 / 技术圈 | 长尾科普 + 雇主品牌 + 招聘 | 长答 + 专栏：方法论、技术拆解、医学 AI 解释 | 月度专栏 + 按需邀答 | 营销味问答、自我吹捧式回答 |
| 抖音 | 医生（碎片场景）、医学生 | 产品认知（视频化） | 30–60s：产品演示、医生使用片段（无真人代言） | 2–3 条/月 | 真人医生喊麦、医患剧、医美引流 |
| 小红书 | 医学生 / 年轻医务工作者 | 雇主品牌 + 招聘（**不卖产品**） | 笔记：办公室、活动、企业文化、医学科普 | 1–2 篇/周 | 病例展示、医美关联、消费者向种草 |
| 视频号 | 公众号留存粉丝 + 下沉 | 公众号长文的视频摘要 | 60–90s 摘要 | 与公众号长文同步 | 重新拍摄真人医生 |

**铁律**：

- **小红书永远不做产品功能演示**——受众错位，且与"医美/养生"标签关联会污染品牌
- **抖音 / 视频号 / 小红书永远不出现真人医生作为代言形象**——医师代言合规风险（参考 §6.2）
- **微博不蹭社会热点 / 节日 / 明星话题**——医学 AI 品牌不参与公共娱乐叙事
- **LinkedIn EN 在本任务范围内仅保持月度更新**，主动获客拆为独立海外推广任务

## 2. 跨平台共享原则（继承 03 主语音）

下列原则在**所有 8 个渠道**无差别生效。每条与 03 章节交叉引用：

1. **中文为主、英文为辅**（03 §1.1）—— LinkedIn EN 例外，详见 §3.3
2. **4 条原则全适用**（03 §3）—— 严谨而非冷漠 / 透明而非啰嗦 / 医生在环 / 不夸大不胆怯
3. **第二人称用"你"不用"您"**（03 §7.4）
4. **数据 claim 带年份与来源**（03 §7.1）—— 平台字符限制下若无空间附 footnote，则改用相对表达（"经 PITL 网络复核"）而非裸数字
5. **"24h" 措辞规则**（03 §3.4）—— 全平台同样适用：无 `(n=X, 来源)` 注释则改写为"敏捷"
6. **禁词全集生效**（03 §6.1–6.3）—— 包括"零幻觉""行业领先""革命性""黑盒"等
7. **不允许 AI 生成医学影像**（FTC 红线，沿用母品牌 v1.1）
8. **不允许真人医生代言**——含视频出镜、口播、签名照、引用医生姓名 + 单独肖像（详见 §6.2）
9. **emoji 在 UI 与正式材料禁用**（03 §7.7）—— 社交平台例外见 §3 各章节
10. **VGI / PITL / AIO 首次出现拼写全称**（03 §4.3）—— 单帖字符紧张时可在评论区 1 楼放术语注释

## 3. 各平台附加规则

> 每个子章节只列**该平台特有**的约束。共享规则不重复。

### 3.1 微信公众号

**账号策略**：建议**单一公众号 "梅斯 AI"**，4 产品作为内容 tag 区分；不开 4 个独立公众号。理由：受众交叉度高、运维成本可控、内容矩阵更利 SEO。

**标题规则**：

| Bad | Good |
|---|---|
| 重磅！DE 黑科技，AI 终结医生编码错误 | DeepEvidence 新增 DRG 编码助手：从 ICD-10 条款到历史病例可追溯 |
| 震惊！这个 AI 比专家还准 | PITL 网络在 1,234 例上的复核结果（n=1,234, 2025） |
| 紧急通知 / 内部消息 / 限时分享 | （直述事件，不制造伪稀缺） |

- 标题字数 ≤ 28 字（微信预览裁切阈值）
- 不使用感叹号开头 / 结尾
- 不使用"重磅 / 突发 / 震惊 / 揭秘"等点击诱饵词
- 副标题选填，用于补充术语注释或受众限定

**头图规范**（详见 §7.1）：

- 横向 900×383 或 1080×460
- 配色用产品 hero 渐变（单产品） 或 母品牌渐变（多产品 / 公司议题）
- **必须出现** master brand endorsement signature `由 梅斯健康 出品` 或 `Powered by MedSci Healthcare`
- **不使用** 图库人像照片（继承母品牌 v1.1 摄影政策）

**正文结构**（推荐模板）：

```
[摘要 80–150 字 · 蓝色块或灰色块]
[正文 · 段落 ≤ 5 行 · 段间留 1 行]
[数据卡片 · 用 4 产品色 token，不开新色]
[纳米级引用 [n] · 文末附完整引用列表]
[Endorsement Signature · 由 梅斯健康 出品]
[阅读原文 → 指向官网产品页或白皮书下载，不指向第三方]
```

**评论区策略**：

- 默认精选放行；含"AI 替代医生 / 医患矛盾 / 患者咨询"内容**不放行**
- 涉医学问题统一回复模板：「关于您提到的临床问题，建议线下就医或咨询您的主治医师。本平台不提供个体诊疗建议。」
- 不开"赞赏"功能；不接广告主投放

**广告与转载**：

- 不接外部广告主（除非母品牌团队批准的合作品牌）
- 转载需注明出处；他人转载需联系市场负责人授权
- 与梅斯医生社区平台（兄弟业务）联动需走 §8 审批流程

### 3.2 LinkedIn（CN）

**账号策略**：

- 公司主页 1 个：`梅斯 AI / MedSci AI`
- 高管账号（CEO / CTO / 首席医学官）独立运营，遵循本文档但允许更个人化的笔触
- 高管账号转发公司主页内容时，**必须加自己的 1 句以上点评**，不允许纯转发

**帖文规则**：

- 字数：1500 字符内（LinkedIn 折叠阈值约 200 字符，重要信息前置）
- 段落：单段 ≤ 3 行；段间留空行
- hashtag：每帖 3–5 个，从下面白名单选：
  - 通用：`#循证医学` `#医学AI` `#PITL` `#VGI` `#临床决策支持`
  - 产品：`#DeepEvidence` `#SeekEvidence` `#DeepInsight` `#RCP`
  - 行业：`#医疗AI` `#数字医疗` `#药企数字化` `#KOL`
- **不**使用泛化标签如 `#AI` `#tech` `#innovation`

**类型化模板**：

| 类型 | 模板要点 |
|---|---|
| 招聘帖 | 岗位 + 团队介绍 + 1 句技术挑战 + 不写薪资范围（除非市场对标公开数据）+ 简历投递邮箱 |
| 行业评论 | 引述公开报告或权威观点 + 我们的视角 1–2 段 + 提出可验证的问题（不强行卖货） |
| 产品里程碑 | 数据 claim 带 `(n=X, 来源)` + 截图 / 卡片 + 阅读原文链回官网 |
| 活动 / 会议 | 时间地点 + 我方主讲嘉宾 + 议题 + 是否有 demo 体验位 |

**禁忌**：

- ❌ 私信群发模板（即使是 BD 也走 §8 渠道）
- ❌ 在公司主页转发任何医患矛盾、社会公共事件、明星议题
- ❌ "thought leadership"式空洞 hot take（无证据支撑的形容词云）

### 3.3 LinkedIn（EN）

**范围限制**：本文档当前批次定义"**存在感保留模式**"——**不主动展开海外获客**，海外推广独立立项。

**当前阶段允许的内容**：

1. 月度精选：从中文公众号长文中挑 1 篇高质量稿件，做 80–150 词 EN 摘要 + 链回 CN 原文（注明 "Available in Chinese"）
2. 季度 highlights：产品里程碑、融资、合规认证（如有），仅事实性发布
3. 学术合作 / 国际会议：参展 / 讲者公告
4. 招聘：仅"open to international candidates"职位

**风格规则**：

- 默认采用 03 §3 四条原则的英文版
- Endorsement signature 用英文版："Powered by MedSci Healthcare"
- 术语优先英文原词：PITL / VGI / AIO / Nano-Citation / CoT / Diff View
- 不做大量整篇翻译再发（"翻译稿"在 LinkedIn EN 受众中识别率高，反而稀释品牌）
- hashtag 用英文：`#EvidenceBasedMedicine` `#MedicalAI` `#PhysicianInTheLoop` `#ClinicalAI`

**升级路径**：当海外推广任务正式立项时，本节升级为独立子文档 `03b-overseas-voice.md`（届时通过 RFC 走 §6 governance 流程）。

### 3.4 微博

**账号策略**：

- 公司认证账号 1 个
- 不开高管个人微博官方关联
- 不开各产品独立微博号

**帖文规则**：

- 字数：140 字内是首屏；超长正文用"长图卡"形式（参考 §7.1 卡片规范）
- 转发原创比 ≤ 3:7（即原创不少于 70%）
- 转发要附 1 句以上自己的观点

**实时反应模板**（按事件类型）：

| 事件 | 是否回应 | 回应方式 |
|---|---|---|
| 学术成果发布（同行 / 自家） | 是 | 引用 + 1 句价值评论；不带任何"领先 / 超越"对比措辞 |
| 行业政策（CDE / NMPA / 卫健委） | 是 | 客观转述 + 我们的相关产品线索（克制，不蹭流量） |
| 公共卫生事件（疫情 / 地震 / 灾害） | 沉默或哀悼 | 不做产品关联；遵循 §5 危机手册 |
| 哀悼日（清明 / 5.12 / 12.13） | 默哀模式 | 黑白头图 / 不发任何商业内容 |
| 同行负面新闻 | 沉默 | 不评论、不转发、不暗讽 |
| 医患纠纷 | 沉默 | 任何评论都有合规风险 |
| 社会娱乐热点 | 沉默 | 与品牌无关 |

**长图卡规范**：1080×1920，使用品牌色 token，不开新色；信任徽章必须出现 1 次；若数据声明则带 `(n=X, 来源)`。

### 3.5 知乎

**答主签名（公司账号或官方答主）**：

> 梅斯 AI · 验证型生成式智能 · medscihealthcare.com

**不做**：硬广式签名（`点击购买` / `添加微信`）。

**邀答策略**：

| 邀答领域 | 是否答 |
|---|---|
| 循证医学方法论 | ✅ 主动答 |
| 临床决策支持系统 | ✅ 主动答 |
| 医学 AI / 大模型在医疗的应用 | ✅ 主动答 |
| SCI 写作 / 选题 / IIT 设计 | ✅ 主动答（SE 主场） |
| 药企数字化 / KOL 触达 / 医学事务 | ✅ 选答（DI 主场） |
| 医生体验 AI 的实操 | ✅ 选答（不出现真人医生姓名 / 头像，只用产品截图） |
| 个体诊疗咨询 | ❌ 不答 |
| 医美 / 养生 / 预防保健 | ❌ 不答 |
| 公司估值 / 融资八卦 | ❌ 不答 |
| 同行对比 / 评测 | ❌ 不答（监管医疗背景下 Legal 邻接，详见 §5.4） |

**回答格式**：

- 数据 claim 带 `(n=X, 来源)`

- **不允许**在回答正文中插入产品购买链接；可在结尾用一行"延伸阅读：[官网产品页]"

### 3.6 抖音

**内容类型白名单**：

1. 产品 demo 录屏（30–60s，配文字字幕）
2. 公司活动片段（团建 / 学术会议布展 / 高校招聘）
3. 学术议题动效解释（PITL 是什么 / VGI 是什么 / Nano-Citation 是什么）
4. 招聘宣传

**禁忌**：

- ❌ 真人医生出镜代言、口播、推荐
- ❌ 医患剧情 / 病例情景剧
- ❌ "这个 AI 比医生准"式对比叙事
- ❌ 配乐选用流量神曲（违反母品牌 v1.1 §9 "克制"原则）
- ❌ 医美引流、保健品引流、药品 CTA

**资产规则**：

- 视频中**信任徽章不可被裁剪**——首尾各出现至少 1 次，且不被字幕/边框遮挡
- 字幕用 PingFang SC / 思源黑体，主色 `#FFFFFF` + 描边色为产品主色
- 不带未授权背景音乐；优先使用平台无版权曲库或自制 BGM

**评论区**：

- 涉医学咨询统一回复："本视频为产品演示，不构成诊疗建议。请就医咨询您的主治医师。"
- 涉合规质疑（"是不是又一个 AI 噱头"等）：可由市场负责人 + 法务双签后回复，不允许单点回应

### 3.7 小红书

**核心定位**（v0.2 修订 · 2026-05-07）：**雇主品牌 + 招聘 + 产品教育**——可做**机制 / 操作 / 信任层演示**与**医生使用介绍**（继承 §6.2 真人医生政策）；仍**不做**消费医疗向种草。

**内容类型白名单**：

1. 办公环境 / 团建 / 节日
2. 招聘故事（员工视角，非真人医生 / 非真实病例）
3. 医学科普（中性、无产品引流）
4. 行业活动签到
5. **产品机制 / 操作 / 信任层演示**（v0.2 新增）—— UI 操作流 / Diff View / Verify It 体验 / 信任徽章解释 / Nano-Citation 工作机制；纯工程化、教育向，不构成治疗推荐
6. **医生使用介绍**（v0.2 新增 · 继承 §6.2）—— 医生作为产品使用者演示操作流程（不是代言；详见 §6.2 二分规则）

**禁忌**（保留 + 新增护栏）：

- ❌ 病例展示、临床场景模拟
- ❌ "医美 / 减肥 / 产后修复"标签关联
- ❌ 真人医生**代言**（出镜口播 / 推荐 / 单独肖像 + 产品 logo——见 §6.2）
- ❌ "种草体"营销话术（"姐妹们冲 / 直接拿下"等）
- ❌ 产品演示与"医美 / 养生 / 减肥"话题串场或同栏目位
- ❌ 演示后立即接产品 CTA / "立即购买" / 直营导流
- ❌ 患者出镜（即使匿名化处理）
- ❌ 推荐特定药品 / 治疗方案

**算法分类风险管理**：

- 内容标签必须用品牌相关词（「医学 AI」「循证医学」「药企数字化」等），避免"医美 / 减肥 / 养生"算法标签污染
- 视频号 / 抖音的 demo 与小红书 demo 应有内容差异化（避免重复推同一 demo）
- 评论区禁医学咨询答复（继承 03a §3.6 抖音规则）

**封面规范**：

## 5. 危机与敏感场景手册

社媒在医学 AI 领域是高合规风险渠道。下表覆盖 6 类典型场景，每类给"沉默 / 等待 / 改时间表 / 主动发声"四档应对建议。最终决策权归市场负责人 + 法务（详见 §8）。

### 5.1 被引文献撤稿

| 阶段 | 行动 |
|---|---|
| T+0（发现） | 内部群通报；运营冻结所有引用该文献的帖文排期 |
| T+24h | 评估影响范围（哪些帖文 / 产品 / 客户报告引用） |
| T+72h | 在每条受影响帖文添加更新声明（不删帖）+ 公众号发"勘误公告" |
| T+1 周 | PITL 网络复核相关结论；如结论本身需修正，再发"修订公告" |

### 5.2 监管公告（CDE / NMPA / FDA / 卫健委）

| 公告性质 | 行动 |

| 行业警示 / 整改 | 沉默 24h 等公告稳定；如涉自家产品线则走 PR + 法务流程，不在社交媒体首发回应 |
| 直接点名 | 不在社交媒体回应，走官方新闻稿 + 法务函渠道 |

### 5.3 公共卫生 / 自然灾害

| 事件性质 | 行动 |
|---|---|
| 重大疫情 / 灾害 | 全平台暂停商业内容 24–72h；如有捐赠 / 援助行动可走 PR 通道发布，不在社媒造势 |

### 5.4 同行 / 竞品

**默认沉默**。监管医疗背景下任何对比声明都有 Legal 风险。仅以下两种例外：

1. **同行重大学术成果**——可发 1 条祝贺帖（不加任何"我们也在做""我们更好"暗示）
2. **同行明显违规事件**——保持沉默，由行业协会 / 监管发声，我们不评论

- ❌ 对比表（"梅斯 AI vs X 公司"）
- ❌ "我们没有 X 公司的问题"
- ❌ 私下转发同行负面新闻
- ❌ 在评论区暗讽

### 5.5 医患纠纷新闻

**铁律：沉默**。任何评论都有合规风险。运营如收到媒体询问，统一回复"以官方新闻稿为准"，转 PR 处理。

### 5.6 自家产品出 bug / 误导输出

### 6.2 真人医生政策（v0.2 修订 · 2026-05-07）

**核心二分**：法务区分两个概念——

- **代言（endorsement）** = 医生以"权威背书"形象向公众**推荐**使用产品 → 合规风险高（医师代言医疗广告法律风险）→ **全平台禁止**
- **使用介绍（user-walkthrough）** = 医生作为产品**使用者**演示操作流程，技术性、流程性，不构成对公众的推荐 → 合规可控 → **有条件允许**

**🚫 全平台禁止（代言）**：

- 视频代言口播 / "我推荐你 / 我建议你 / 你应该用"型推荐语
- 单独肖像 + 产品 logo / 姓名 + 单独肖像组合
- "X 医生推荐"标题
- 与产品 CTA 紧邻的医生形象（即使内容不是代言，组合后构成代言）

**✅ 有条件允许（使用介绍）**：

医生作为产品使用者演示操作流程，**必须同时满足以下全部**：

1. 内容是"我如何使用"而非"我推荐你使用"——视角是使用者，不是背书者
2. 不出现"推荐 / 建议你买 / 你应该使用 / 我证明这个准"等推荐语
3. **不与产品 CTA 紧邻**——演示后不能立刻接"立即购买" / "联系销售" / 直营导流；CTA 须放在另一段落且无医生形象
4. **医生身份披露真实**——医院 + 职级 + 执业范围（不允许虚构 / 借用其他医院职级）
5. **演示内容限定于产品操作流**——UI 操作 / 输入查询 / 看返回结果 / 触发 PITL；**不延伸**为治疗建议、处方推荐、个体诊断
6. **文末附声明**："个人使用经验，不构成医疗推荐 / 不构成对患者诊疗建议"
7. **医生书面同意**——参与录制前签署知情同意书（含转载范围 / 撤回机制 / 有效期 / 肖像权授权范围）；同意书 SourceID 进实证库（参 03f §1.2 客户案例 claim）
8. **不与"医美 / 养生 / 减肥"话题串联**

**仍允许**（继承 v0.1）：

- 医生作为活动合影中的一员（无单独突出）
- 引用医生姓名 + 学术职务（无肖像）作为"专家咨询团"列表
- PITL 网络的合作医生**仅以匿名化数据**（`n=X 位医生`）出现

**仍禁止**（继承 v0.1，不因允许"使用介绍"而松动）：

- 任何形式的医生**代言** / 推荐
- 患者出镜（即使匿名化处理）
- 推荐特定药品 / 治疗方案
- "医美 / 养生 / 减肥 / 产后修复"产品的关联

**边界判定灰区**：法务出"判定指南"作为 v0.2.1 patch（含案例库；例如"医生说我每天用 DeepEvidence 查文献"算介绍还是代言）。本批次先把"使用介绍 vs 代言"二分规则写入条款。

**理由**：

- 代言禁忌——医师代言医疗广告在中国与多数司法辖区合规风险高；与 PITL "网络化匿名复核"机制的品牌叙事一致
- 使用介绍允许——技术性 / 流程性内容不构成对公众的推荐；可在合规护栏下作为产品教育素材，丰富社交内容形态（特别是抖音 / 视频号 / 小红书）
- 八条护栏共同构成"内容是教育 vs 推销"的合规判定底线

### 6.3 数据图表配色

- 严格使用 `ai-tokens.css` 与 `colors_and_type.css` 的 token，不开新色
- 多产品对比图：每产品色为 dataset 主色
- 时间序列图：用母品牌主色 + 单产品色 accent
- 误差棒 / 阴影：用 `--neutral-300` 系
- success 状态用绿色仅限 `--success` token；error 状态用红色仅限 `--error` token

### 6.4 信任徽章出现规则

- **每篇长文 / 视频至少 1 次**信任徽章可见（PITL / AIO / VGI 任一）
- 徽章不能被裁剪、不能被叠图遮挡、不能改色
- React 组件直接用 `react/src/trust/` 下三个组件（PITLBadge / AIOBadge / VGIMark）

### 6.5 产品色混用规则

**铁律：单帖单产品 scope 或母品牌 scope，不混用**。

- 如发文涉及多产品，使用母品牌色作主调，4 产品色仅作小面积 accent
- 不在同一卡片同时使用 DE `#2563EB` + SE `#9333EA` + DI `#0891B2` + RCP `#0F62FE` 作为面积色
