## 0. 适用范围与边界

### 0.1 覆盖

本文档覆盖以下场景：

- 任何以 `@medsci-ai.com` / `@yogyoung-code` / 未来 `@medsci.com.cn` 等品牌域名发出的**对外邮件**
- 任何代表"梅斯 AI" / "MedSci AI"伞标的对外邮件（即使从员工个人邮箱发出）
- 涉及对外承诺 / 数据 claim / 客户合同 / 撤稿响应等**需 audit trail 的内部邮件**
- 营销 EDM / 客户通讯 / onboarding 流程 / 续约提醒 / 客户成功案例 / 媒体公关 / 监管通知 / 紧急召回

### 0.2 不覆盖

- **员工个人非 work 邮件**——员工以个人身份发出的非品牌邮件
- **母品牌（梅斯健康）非 AI 子品牌邮件**——按母品牌 v1.1 邮件规范
- **兄弟业务（梅斯医生）邮件**——独立邮件域 + 独立规范
- **Slack / Teams / 微信 / 飞书 IM**——按 03 / 03a 处理
- **SMS / 短信**——独立渠道（短信合规与邮件不同）；待独立子文档（03g 占位 · 暂未起草）
- **API webhook 通知**——纯系统对系统，按 06 §5.4 + 工程规范

### 0.3 优先级层级

冲突时按以下顺序裁决：

1. 03f §3.4 撤稿响应铁律（紧急邮件场景）
2. 03 §1 主语言（CN 主 / EN 仅特定场景）
3. 03 §6 禁词清单
4. 03a §6.2 真人医生政策
5. 03e §6.1 大区独发 + 事后 review（区域化邮件）

## 1. 邮件分类与签字门槛矩阵

### 1.1 6 大类邮件

| # | 分类 | 描述 | 频次 | 签字门槛 |
|---|---|---|---|---|
| **C1** | **事务性 (Transactional)** | 系统触发：注册确认 / 密码重置 / 订单 / 发票 / 申请回执 / API 状态 | 每用户多封 | Tech Lead 单签（系统）+ 法务一次性 review 模板 |
| **C2** | **营销 (Marketing / EDM)** | 主动外发：产品发布 / 活动邀请 / 行业 newsletter / 节日问候 | 每周 / 月 | 市场 + 法务双签**每个模板**；活动级 Sponsor 信息周知 |
| **C3** | **客户支持 (Customer Support)** | 1:1 回复：问题答复 / 故障通知 / 工单跟进 | 按需 | 客户支持 lead 单签（在脚本框架内）+ 法务异步 review 模板 |
| **C4** | **内部协同 (Internal)** | 跨团队 / 跨部门协调：项目状态 / 决议通知 / 周会纪要 | 每日 | 发件人自审；如涉对外承诺需双签（按 06 §3）|
| **C5** | **媒体 / 公关 (PR)** | 媒体回复 / 采访邀请 / 新闻稿邮件 | 按需 | **PR + 法务双签**；高合规事件（撤稿 / 监管）+ Sponsor |
| **C6** | **紧急 / 撤稿响应** | 撤稿声明 / 数据偏差更正 / 监管告警 / 安全召回 | 罕见 | **三签**（市场 + 法务 + Sponsor）+ 24h 内发出（继承 03f §3.4）|

### 1.2 跨产品 / 多产品邮件

涉及 ≥ 2 个产品（DE / SE / DI / RCP）的邮件：

- **签字门槛上调一级**（如本来 C2 双签，变三签 + Sponsor）
- 必须明确**主推产品**与**附带产品**——不允许 4 产品平等并列（防止信息淹没）
- 视觉 BrandScope 用主推产品色

### 1.3 大区独发（继承 03e §6.1）

- 大区市场可独发 C2 营销邮件 + C4 内部协同
- 不可独发 C5 媒体 + C6 紧急
- 事后 100% review by 总部 + 法务（7 工作日内）
- 3 次违规取消独发权

### 1.4 客户共发布（合作伙伴邮件）

涉及客户名称 / case study 的邮件：

- **必须 §10 客户书面确认**（不限于发件前最后 24h）
- 客户撤回权（任何时间，1 个工作日生效）
- 客户姓名 / 头衔 / case 数据均按 03f §2 SourceID 注册

- 如有大规模伪造事件触发 06 §2.4 紧急修订流程

## 3. 主题行规则

### 3.1 通用规则

- **长度 ≤ 50 字符**（移动端邮箱默认 ~40 字符截断）
- **不用全大写**（"重要！" 全大写在中文也是垃圾邮件红旗）
- **不用过多 emoji**（继承 03a · 默认禁 + 白名单：🎉 / 📌 / ⚠️ 共 3 个 · 仅营销 + 紧急）
- **不用 [Re:] [Fwd:] 假装回复**（钓鱼手段，监管禁忌）
- **不用 "紧急" / "限时" / "免费" / "100%"** 等反垃圾邮件触发词

### 3.2 主题行禁词清单

继承 03 §6 禁词；额外邮件特异禁词：

- "紧急" / "URGENT" —— 仅 C6 撤稿响应允许
- "限时" / "Limited time" —— 仅有具体时间窗 + 法务备案的促销允许
- "免费" / "FREE" —— 不允许（医疗领域监管敏感）
- "保证" / "Guaranteed" —— 不允许（医疗领域 FTC 禁忌）
- "100%" / "全部" / "彻底" —— 不允许（监管对绝对化用语限制）
- "AI 大脑" / "智能大脑" —— 继承 03 §6 禁词
- "终结 AI 幻觉" —— 继承 03 §6 禁词

### 3.3 主题行模板

| 邮件类型 | 模板 | 示例 |
|---|---|---|
| 事务性 | `[产品][动作完成]` | `[DE] 注册成功 · 你的循证查询额度已开通` |
| 产品发布 | `[产品][版本/功能][价值钩子]` | `DeepEvidence v2.3 · 30 秒拿到带证据的临床决策` |
| 活动邀请 | `[活动名][日期][受益]` | `临床循证 AI 实战工作坊 · 5/15 上海 · 名额有限` |
| 客户成功案例 | `[客户类型][结果数字][产品]` | `三甲医院呼吸科：DRG 编码错误率从 8% 降至 0.6%` |
| 撤稿响应 | `⚠️ 数据修正声明 · [文献标题简述] · [日期]` | `⚠️ 数据修正声明 · "AI 辅助诊断准确率 91.3%" · 2026-05-08` |
| 续约提醒 | `[产品]订阅[时间窗]即将到期 · 续约享[优惠]` | `DeepEvidence 订阅 30 天后到期 · 续约享老客户 9 折` |

### 3.4 A/B 测试规则

- 仅 C2 营销允许 A/B 测试主题行
- 单次测试样本 ≥ 1000 收件人
- 测试维度：版本数 ≤ 2（A/B；不允许 A/B/C/D 多变体降低统计显著性）
- 法务**不参与** A/B 测试主题行（每个版本主题行需法务一次性 review；不需逐次 review）
- 结果留痕：实证库 SourceID 注册（参 03f §2）

## 4. 正文结构

### 4.1 称呼

- **CN**：默认"你"（参 03 §2.1）；不用"您"
- **EN**：默认"You"
- **B2B 客户合同 / 法务 / TOS / 隐私政策**：用"您"（继承 03 §2.1 例外）

不用：

- "亲爱的 X 先生 / 女士" —— 过于正式
- "Dear Doctor" —— 过于美式
- "Hi there!" —— 过于美式 + emoji 化

推荐：

- "[姓] 医生，你好" / "Dr. [Last name],"
- "[名字]，你好" / "Hi [first name],"（已知熟悉关系）
- "你好" / "Hello"（无个性化但不显得冷漠）

### 4.2 段落与句长

- 段落 ≤ 4 段
- 每段 ≤ 3 句
- 单句 ≤ 25 字（CN）/ ≤ 20 词（EN）
- 段落之间用空行（HTML 邮件用 `<p>`，纯文本邮件用空行）
- 移动端友好：避免长段落 / 大表格 / 复杂排版

### 4.3 endorsement signature 在邮件中的呈现

| 邮件类型 | endorsement 出现位置 | 颜色 | 字号 |
|---|---|---|---|
| 事务性 / 营销 / 客户支持 / 1:1 业务 / PR | **必须**——在邮件签名档底部 | `#666666`（继承 02 §2）| 12px（桌面）/ 14px（移动）|
| 内部协同 / 法务 | 可省（员工知情）| — | — |

### 4.4 信任徽章 / VGI / Nano-Citation 在邮件中

- **PITLBadge / AIOBadge / VGIMark**：仅在产品介绍 / case study 邮件出现；不滥用
- **Nano-Citation [n]**：邮件 body 内数据 claim 必带 [n]（继承 03f §2）；[n] 不可点击的纯文本邮件版本——用 `[1]`，对应链接放在邮件底部"数据出处"段
- **CoT 动画**：邮件中**不可用**——邮件不支持 CSS 动画（GIF 替代会很大且违 04c §3.4 装饰禁忌）；用静态 spinner 占位图代替

### 4.5 CTA（呼吁行动）

- **每封邮件 ≤ 2 个 CTA**——主 CTA 突出（按钮 / 链接）；次 CTA 文字（"或访问帮助中心"）
- CTA 文字遵守 03 §3.3 命名规范：
  - DE：`查证一下 / Verify It / 看 Diff`
  - SE：`看看选题 / 构造检索式 / 草拟一稿`
  - DI：`雷达 + 我的产品 / 看 Rising Star`
  - RCP：`发起任务 / Run a pulse / 看任务反馈`
- CTA 链接必带 UTM 参数（`utm_source=email&utm_medium=[email-id]&utm_campaign=[campaign]`）
- CTA 不允许"点这里" / "立即购买" / "马上抢"——降低专业感

### 4.6 emoji 使用（继承 03a · 邮件特异）

继承 03a §6.5 emoji 白名单 + 邮件特异规则：

- C1 事务性：**完全禁用** emoji
- C2 营销：白名单（🎉 产品发布 / 📌 重要通知 / ⚠️ 撤稿声明）—— 每邮件 ≤ 2 个
- C3 客户支持：仅在客户先用 emoji 时回 1 个（镜像礼貌原则）
- C4 内部协同：员工自由（仅本子文档不约束）
- C5 PR / 媒体：完全禁用
- C6 撤稿响应：**仅 ⚠️**（一次性，主题行）

### 4.7 链接与追踪

- 所有外链必带 UTM 参数
- 短链工具（如 bit.ly）—— **不允许**（监管反钓鱼判定为可疑 + 不可追踪）
- 内部链接（落地页 / 帮助中心 / 客户后台）按域名锁定 `*.medsci-ai.com`（v0.x.x）
- 客户邮件中**不允许**含 PII 的查询参数（如 `?email=customer@example.com`——会被 referer 泄露）；用 token 化 ID

## 5. 数据 claim 与引用（继承 03f）

### 5.1 引用格式

继承 03 §7 + 03f §2 SourceID：

| 场景 | 邮件正文 | 邮件底部 |
|---|---|---|
| 营销 / 产品介绍 | `91.3% (n=1,234, 2025)¹` | "数据出处" 段：`¹ EVID-2025-0042 · [研究题目] · [DOI / 链接]` |
| 客户成功案例 | `XXX 医院呼吸科 · DRG 编码错误率从 8% 降至 0.6%¹` | `¹ EVID-2026-0123 · 客户授权使用 · [客户落地页]` |
| 撤稿响应 | `原 91.3% 修正为 87.6%（n=1,180, 2025）¹ · 详见 §修正说明` | `¹ EVID-2025-0042 · 已撤稿；EVID-2026-0987 · 修正版分析` |

### 5.2 不可在邮件中使用的 claim

继承 03f §1.2 不允许 claim：

- L1 仅公开数据但未 SourceID 注册的 claim
- L2 内部数据但未脱敏的 claim（含客户 PII）
- 维基百科 / 百度百科作为唯一来源
- 已撤稿研究（见 03f §3）
- AI 生成综述未核验

### 5.3 跨场景一致

继承 03f §5.1：邮件中的 claim 必须与官网 / 落地页 / 白皮书 / 社交媒体 / UI 一致；不允许"邮件营销端放大"。

## 6. 真人医生政策（继承 03a §6.2 · 邮件特异）

### 6.1 邮件中医生形象的两类处理

| 维度 | 邮件 body 内 | 落地页 / 附件 |
|---|---|---|
| **医生姓名 + 头衔（文字）** | ✓ 允许（在 case study / 学术合作邮件中）| ✓ 允许 |
| **医生头像照片** | ❌ **不允许** | ✓ 允许（按 04c §2.3 + 03a §6.2 8 条护栏）|
| **医生视频 / GIF** | ❌ **不允许** | ✓ 允许（同上）|
| **医生引语（"使用心得"）** | ✓ 允许 + 必须 §6.2 8 条护栏 | ✓ 允许 |

### 6.2 邮件中包含医生信息的 8 条护栏（继承 03a §6.2）

每封涉及真人医生的邮件必满足全部 8 条：

1. ✓ 医生**主动**书面同意（不限于发件前最后 24h；客户撤回权 1 工作日）
2. ✓ 引语**真实可考**（实证库 SourceID 注册）
3. ✓ 不与商业 CTA**紧邻**（医生引语段与"立即购买" CTA 间隔 ≥ 1 段）
4. ✓ 文末免责声明：`本引语为该医生个人使用心得，不构成处方建议或代言推荐`
5. ✓ 不含 ROI / 财务回报数据
6. ✓ 标注医生头衔时**不夸大**职位（"主任医师" ≠ "首席专家"）
7. ✓ 引语限定**操作流程**（"通过 PITL 验证" 而不是 "AI 替我看病"）
8. ✓ 法务一次性 review 模板 + 每次发送前快速核对

### 6.3 患者出镜（继承 04c §2.3）

- **邮件 body 内永远不允许**患者头像 / 真名 / 病例细节
- 即使匿名化也不允许（邮件转发链不可控）
- 患者 case 必须落到落地页（按 04c §6 W2 交付前过渡期政策）

## 8. 退订与合规

### 8.1 营销邮件必含退订链接

法律要求（**所有营销 EDM 必含**）：

- **PIPL §17**（中国个人信息保护法）—— 营销同意需独立、明示、可撤回
- **CAN-SPAM Act**（美国）—— 退订链接必须 ≤ 1 个 click + ≤ 10 工作日生效
- **CASL**（加拿大）—— express consent + opt-out 任何时间
- **GDPR Art. 21**（欧盟）—— 退订权 + 数据可携权

### 8.2 退订规范

| 维度 | 规则 |
|---|---|
| 退订链接位置 | 邮件底部 + 字号 ≥ 10px（不藏） |
| 退订链接文字 | "退订" / "Unsubscribe"（不用"取消订阅" / "停止接收"等模糊词） |
| 退订流程 | ≤ 1 个 click 完成 + 不需登录 + 不需 password |
| 退订生效时间 | ≤ 1 个工作日（远低于 CAN-SPAM 的 10 工作日要求）|
| 退订确认 | 不需"二次确认"（违反 PIPL）；点击即生效，发确认邮件 |
| 退订列表保留 | 7 年（与撤稿响应 trail / HKEX 上市公司文件保留要求对齐 · 03f §6）|

### 8.3 事务性邮件不需退订但需"为何收到"段

事务性邮件（C1）法律豁免退订要求，但**必须**在邮件底部包含：

```
你收到本邮件因为你是 梅斯 AI · DeepEvidence 注册用户（注册时间 2026-04-12）。
本邮件为 [事务性类型]，无法退订；如需停用账号请访问 [设置页面链接]。
```

### 8.4 个人信息保护

继承 03f §2 + 04c §2.5 PII 模糊化：

- **不在邮件 body 中传 PII**（手机号 / 身份证 / 病例编号 / 真实诊断细节）
- 客户姓名仅用 `[姓] 医生` / 单字姓名 + 头衔
- 客户公司用工商登记简称（不用全称 + 地址）
- 用 SourceID 而非客户姓名追踪 case

### 8.5 营销邮件双 opt-in（高合规推荐）

虽 PIPL 不强制双 opt-in，但医疗领域推荐双确认机制：

1. 用户注册时勾选"接收营销邮件"复选框（默认**不勾选**——continued opt-in is OK; 不允许 pre-checked）
2. 系统发出"确认订阅"邮件（事务性 C1）
3. 用户点击"确认"后才进入营销名单

## 11. 附件规范

### 11.1 命名规则

强制格式：`[YYYYMMDD]-[产品]-[类型]-[版本].[扩展名]`

示例：

- ✓ `20260508-DE-技术资料-v0.1.pdf`
- ✓ `20260508-RCP-客户合同模板-MSA-v2.docx`
- ✓ `20260508-DI-鉴权资料-INTERIM-2026-0042.pdf`

错误示例：

- ❌ `final-final.pdf` —— 无版本信息 / 无追溯
- ❌ `机密.pdf` —— 文件名暴露敏感性
- ❌ `客户A数据.xlsx` —— 文件名含 PII（即使匿名化）
- ❌ `~副本(1).pdf` —— Office 临时文件命名

### 11.2 PII 与水印

涉客户的附件（合同 / 提案 / case study）：

- 在每页页眉添加 `供 [客户公司简称] · [日期] · 仅供参考` 水印
- 水印颜色 `#999999` · 字号 8pt（不影响阅读但暴露泄露源）
- PDF 加 metadata：`Created by MedSci AI · [SourceID] · [Date]`
- 不在文件名中暴露 PII —— 用 SourceID 替代客户姓名

### 11.3 大小限制

- 单附件 ≤ 10 MB（多数邮件服务商上限 25 MB；保守取 10 MB）
- 多附件总和 ≤ 25 MB
- 超过 → 用内网链接 / GitHub Packages / 临时素材库（与 04c §4 临时素材库基础设施复用）

### 11.4 危险附件类型

**永远不允许**作为邮件附件：

- `.exe` / `.bat` / `.sh` / `.app` —— 可执行文件（监管 + 反钓鱼）
- `.zip` / `.rar` / `.7z` —— 压缩包（除非明确告知收件人 + 单文件解压）
- `.iso` / `.dmg` —— 镜像（同上）
- 含动态宏的 `.docm` / `.xlsm` / `.pptm` —— 用 `.docx` / `.xlsx` / `.pptx` 替代

### 11.5 加密附件

涉客户合同 / 法律文件 / 财务数据的附件**必须**密码加密：

- PDF 用 256-bit AES（Adobe Standard）
- 密码**不在同一封邮件**发送（必须电话 / 短信 / IM 单独发）
- 密码长度 ≥ 12 字符 + 含大小写 + 数字 + 符号
