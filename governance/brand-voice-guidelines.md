## 1. 品牌基础

### 1.1 主语言

- **CN 主**——产品 UI / 营销网站 / 社交媒体 / 用户可见的错误/异常/空态文案
- **EN 辅**——投资者 / 出海 / API 技术文档 / 代码层标识符
- 判断标准：**该字符串是否会被翻译给用户？是→CN / 否→EN**
- LinkedIn EN 仅"存在感保留模式"——海外推广独立立项

### 1.2 品牌定位（必背）

> **梅斯 AI · 验证型生成式智能。让每一句结论可验证，每一证据可溯源。**
>
> **MedSci AI · Verified Generative Intelligence. Every conclusion is verifiable; every evidence is traceable.**

反定位："不替代医师判断，只放大其可靠性。"

### 1.3 命名铁律

| 元素 | 规则 |
|---|---|
| 4 产品名 | DeepEvidence / SeekEvidence / DeepInsight / RCP——**英文 wordmark 形式不翻译**；行内提及保持驼峰 |
| RCP 全称 | "Rapid Clinical Pulse"仅首次提及；wordmark 永远是"RCP"三字母 |
| 内部代号 | "Hippocrates"（=DE 内部代号）**禁止对外** |
| 母品牌正式署名 | "梅斯健康" / "MedSci Healthcare" |
| AI 子品牌伞标 | "梅斯 AI" / "MedSci AI"——**不可拆解**为"梅斯"+"AI"；仅当作伞标整体出现时使用 |
| 兄弟业务 | "梅斯医生"是医生社区平台，与 AI 子品牌严格不互换 |

### 1.4 缩写首次出现拼写

PITL / VGI / AIO / CDS / IIT / SPL 首次出现拼写全称 + 缩写：

- "医师在回路（Physician-in-the-Loop · PITL）"
- "验证型生成式智能（Verified Generative Intelligence™ · VGI）"
- "AI 检索优化（AIO · AI Optimization）"——**不是** "All-In-One"

## 2. 写作微规则

### 2.1 第二人称

- 默认用 **"你"**（CN）/ **"You"**（EN）
- **不**用"您"——例外仅 TOS / Privacy / 法律声明

### 2.2 数字格式

- 千分位半角逗号：`3,300,000`
- 数据 claim **必带年份和来源**：`91.3% (n=1,234, 2025)¹`
- 中英混排英文术语前后**保留半角空格**：`让 AI 与医生协同`（不写"让AI与医生协同"）
- 中文紧凑场景可用"万"："`330 万医生`"，但正式 claim 仍要 `(n, YYYY)`

### 2.3 引用规范

- 上标方括号 `[1]` / `[1, 2]`——**不**用 `(1)` / `*1*`
- 完整引用列表附底部
- hover 触发原文 tooltip

### 2.4 段落 / 篇幅

- UI 文案：单段 ≤ 2 句；说明性段落 ≤ 4 行
- 营销长文：单段 ≤ 5 行；段间 1 行空白
- **不在 UI 中使用感叹号**

### 2.5 emoji 与符号

- **UI 与正式材料禁用 emoji**
- 社交平台**有限例外**（详见 §6.6）
- 不用 ❤️ 💪 🎉 🔥 💯 🚀 ⚡ 等情感符号
- ✓ ◐ 🔒 ⚑ 仅作信任徽章组件内固定 glyph

### 2.6 "24h"措辞特别规则

任何"24h" / "24 小时"措辞必须附 `(n=X, 来源)` 注释；否则改写为"敏捷" / "agile" / "目标: 24h"。

## 3. 禁词全集（**对外材料发布前必扫**）

### 3.1 中英禁词

**中文**：零幻觉 / 无幻觉 / 零错误 / 100% 准确 / 完美准确 / 绝对准确 / 行业领先 / 业内第一 / 行业最佳 / 革命性 / 颠覆性 / 前沿 AI / 最先进 AI / 被数千医生信任 / 广受好评 / 终结幻觉 / 杜绝错误 / 重磅 / 黑科技 / 震惊 / 揭秘

**英文**：`zero-hallucination` / `hallucination-free` / `100% accurate` / `perfect accuracy` / `flawless` / `industry-leading` / `best-in-class` / `world's #1` / `revolutionary` / `cutting-edge AI` / `trusted by thousands`

### 3.2 AI / 医学专属禁词

| 禁词 | 替代 |
|---|---|
| AI 自动诊断 / 自动处方 | AI 辅助诊断（待医生确认）/ 处方建议（待签发）|
| AI 替代医生 / AI 取代专家 | AI 与医生协同提升判断可靠性 |
| 黑盒 / 黑箱 | 算法过程 / 推理链 |

| 大模型 / LLM（患者面向）| —（不暴露技术实现）|
| 无年份无来源的数字 | (n=X, 来源, 年份) |

### 3.3 催促 / FOMO 禁忌（社交 + 营销）

- "立即抢购" / "限时" / "速速" / "点击立得" / "不容错过" / "姐妹们冲" / "直接拿下"

## 4. 数据 claim 实证（**写每一个数字必查**）

### 4.1 实证强度分级（L1–L4）

| 级别 | 定义 |
|---|---|
| L1 | 公开来源（行业报告 / 媒体）|
| L2 | 可审计的内部数据（带 audit trail）|
| L3 | 同行评议研究（RCT / Meta / SCI / 国家级共识）|
| L4 | 官方批文 / 书面合同 / 客户书面同意 |

### 4.2 8 类 claim 最低实证

| Claim 类型 | 最低实证 | 谁审 |
|---|---|---|
| 监管 / 合规 | L4 | 法务+合规 |
| 临床效果 | L3 + L2 | 产品+学术+法务 |
| 产品性能 | L2 | 产品+市场（涉客户名单+法务）|
| 产品功能 | L2 | 产品+市场 |
| 市场规模 | L2 + L4（客户书面同意）| 市场+法务 |
| 行业评论 | L1 | 市场 |
| 学术合作 | L4（书面 MOU）| 学术+法务 |
| 客户案例 | L2 + L4（客户书面同意）| 市场+KAM+法务 |

### 4.3 SourceID 体系（铁律）

- 每条引用源进入对外物料前必须先在实证库注册获 SourceID（格式 `EVID-YYYY-NNNN`）
- 状态必须 **approved** 才可引用（pending / revoked 不可用）
- **不允许静默修改**——改动需"版本号+修订记录"

### 4.4 11 项不允许来源（红线）

1. 维基百科 / 百度百科作为唯一引用
2. **撤稿文献**（Retracted）
3. Predatory journal / 灰色出版商
4. AI 生成的"文献综述"未经原文逐条核验
5. 客户**口头**同意的 case（必须书面）
6. 兄弟业务（梅斯医生社区）数据冒充梅斯 AI 直接交付
7. 同行公司的内部材料
8. 未公开发表的会议摘要（除非随后正式发表）
9. 截屏 / 二次引用
10. 截图来源不可定位
11. 未授权的母品牌资产数据

### 4.5 跨场景一致

- 同一 claim 在白皮书 / 网站 / UI / 销售 PPT / 客户合同里表述**一致**
- 营销与工程对同一数字差异时**以最保守表述为准**
- 中英版本须语义一致——不允许"中文夸张+英文克制"或反之

## 5. 真人医生政策（**社交 + 视频 + 摄影必查**）

### 5.1 核心二分（v0.2）

- **代言（endorsement）** = 医生以"权威背书"形象向公众**推荐**使用产品 → **全平台禁止**
- **使用介绍（user-walkthrough）** = 医生作为产品**使用者**演示操作流程 → **有条件允许**

### 5.2 全平台禁止（代言）

- ❌ 视频代言口播 / "我推荐你 / 我建议你 / 你应该用"型推荐语
- ❌ 单独肖像 + 产品 logo / 姓名 + 单独肖像组合
- ❌ "X 医生推荐"标题
- ❌ 与产品 CTA 紧邻的医生形象

### 5.3 使用介绍 8 条护栏（必须**全部**满足）

1. 内容是"我如何使用"而非"我推荐你使用"
2. 不出现"推荐 / 建议你买 / 你应该使用 / 我证明这个准"等推荐语
3. **不与产品 CTA 紧邻**——CTA 须放在另一段落且无医生形象
4. **医生身份披露真实**（医院 + 职级 + 执业范围）
5. **演示内容限定于产品操作流**——不延伸为治疗建议 / 处方推荐 / 个体诊断
6. **文末附声明**："个人使用经验，不构成医疗推荐"
7. **医生书面同意**（含转载范围 / 撤回机制 / 有效期 / 肖像权授权）+ 同意书 SourceID 进实证库
8. **不与"医美 / 养生 / 减肥"话题串联**

### 5.4 患者形象

- **过渡期一律禁止**（即使匿名化）
- W2 交付后视母品牌规范评估

### 5.5 PII 模糊化

拍摄 / 截屏中可见的 PII 必须模糊化或回避：

- 员工胸牌 / 工号 / 工卡号
- 病例编号 / 床号 / 患者姓名
- 屏幕真实数据
- 名片 / 文件 / 白板

## 7. 中国大陆受众分层（**地理本地化必查**）

### 7.1 医院 / 医生 4 层（v0.2 修订）

| 层 | 定义 |
|---|---|
| **T1** | 一线城市 + 一线医学院附属医院 |
| **T2** | 新一线 / 强二线省会三甲 |
| **T3** | 二三线市级三甲 + T1/T2 区域二级 |
| **T4** | 基层与新业态：T4-A 公立基层（县域+三四线二级）+ T4-B 新业态（民营 / 第三方 / 互联网医院 / 连锁体检）|

港 / 澳 / 台 / 日本 / 新加坡 / 海外华人 → **归 03b 海外推广**（独立立项）。

### 7.2 药企 / 产业方 3 层

- **MNC**（跨国药企）
- **国内大型 Pharma**
- **Biotech / Mid Pharma / CRO / 调研 / AI 公司**

### 7.3 文案分层 5 维

| 维度 | T1/T2/MNC | T3/T4-A/国内大型 Pharma | T4-B/Biotech/CRO/AI 公司 |
|---|---|---|---|
| 术语密度 | 学术英文保留 | 中文先行+英文括注 | 业务结果优先 |
| 引用偏好 | SCI [PMID/DOI] | UpToDate + 中文指南 | 客户 case + 业务数据 |
| CTA 风格 | 克制学术 | 价值前置低门槛 | ROI/速度直述 |
| 时间表达 | 2–4 周（IT 评审） | 4–10 周 | 1–2 周 |
| 政策语境 | DRG 全面铺开 | DRG/DIP 试点 | 不适用 / 互联网医院 |

**铁律**：降低术语密度 ≠ 降低实证级别。T4-A 县域物料每条 claim 仍需 SourceID + 满足 §4.2 最低实证。

### 7.4 跨层 case 引用

- **同层优先**——T1 case 给 T1 客户，T4-A 给 T4-A，T4-B 给 T4-B
- **T4-A 与 T4-B 之间也算跨层**（业态差异大）
- **跨层引用须文中标注差异**："本案例来自 T1 三甲，下沉到 T4 应用时 X 部分有差异"
- **MNC case 严禁挪用给 Biotech**

### 7.5 政策语境（铁律）

- **DRG vs DIP**：永远不在同一份本地化稿件中混用；按目标城市选定一种
- **省级医保**：不在国家级正文引用具体省的支付细则
- **互联网医院**：不能跨省提供"处方建议"型表述；用"用药参考" / "循证查询"替代；**永远附"本输出不构成处方"声明**
- **数据出境**：默认"数据本地化部署 / 训练数据未出境 / 模型推理在境内"；禁忌"全球数据训练 / 海外算力加持"
- **集采 / 一致性评价**：不允许"帮你在医生检索中盖过原研药"暗示性表述

### 7.6 县域内容禁忌（T4-A）

- ❌ 暗示"T4-A 医生水平不足需 AI 弥补"
- ✅ "AI 让基层触达 T1 同等的循证资源"

### 7.7 大区独发权（v0.2 修订）

- **大区可独立发布常规对外内容**（活动 / 团队 / 案例 / 政策评论 / 数据 claim 等所有日常内容）
- **总部 + 法务事后 review**——每周 100% / 7 天内 / 3 次违规取消独发权

## 9. 视觉与 a11y 工程级（**04a v0.2 已签**）

### 9.1 颜色对比度

- **WCAG 2.1 AA 强制**——文本 ≥ 4.5:1 / 大文本 ≥ 3:1 / 非文本 ≥ 3:1
- **AAA 高合规建议**——临床决策辅助 / 招股 / 监管对接产品入口
- DI `#0891B2` 在白底 4.50:1 仅刚达 AA——T4 县域物料字号需加大至 18px / 14px Bold
- Endorsement `#666666` 反白态升至 78% 白（`#C7C7C7`）

### 9.2 不依赖颜色（铁律）

- 信息传递必须有**形状 / 文字 / 位置**冗余标识
- DE `#2563EB` 与 RCP `#0F62FE` 在红绿盲下相近——同界面同时使用必须配合 wordmark / icon
- success / error 必须配合 ＋ / − 符号 + 文字"新增" / "删除"

### 9.3 触控目标

- 移动端 ≥ **44×44 px**（强制）
- 桌面端 ≥ 24×24 px（建议 32×32）
- 信任徽章 hit area 扩展（视觉外 inset -10px）
- Nano-Citation [n] 移动端强制 padding 0.5em

### 9.4 键盘交互

- 所有交互元素 Tab 可达 / Enter / Space 触发 / Esc 退出
- `:focus-visible` 焦点环：2px solid + 2px offset + 产品色
- 不允许 `outline: none` 而无替代焦点指示
- 5 类组件 trap 处理：Nano-Citation 侧边面板 / Disclosure / Diff View 切换器 / CoT / MasterBrandLogo 链接

### 9.5 aria 完整规范

- 装饰性 SVG `aria-hidden="true"`
- 自定义控件必须 aria-label / role / aria-expanded / aria-controls
- 状态消息（CoT 思考）用 `aria-live="polite"`；高合规告警（撤稿）用 `aria-live="assertive"`

### 9.6 RTL 不支持

阿拉伯 / 希伯来语在范围外；如未来纳入需视觉+法务联评。

### 9.7 CI 阈值

- axe-core **0 critical / 0 serious**（任一即 fail CI）
- Lighthouse a11y **≥ 95**
- moderate 警告允许 ≤ 5 但需 ticket 跟踪

## 10. 动效规则（**04b v0.2 已签**）

### 10.1 缓动与时长 token

| Token | 值 |
|---|---|
| `--ease-out` | cubic-bezier(0.0, 0.0, 0.2, 1) — 入场 |
| `--ease-in` | cubic-bezier(0.4, 0.0, 1, 1) — 退出 |
| `--ease-in-out` | cubic-bezier(0.4, 0.0, 0.2, 1) — 切换 |
| `--ease-linear` | cubic-bezier(0, 0, 1, 1) — 进度条 |
| `--duration-micro` | 100ms — hover/pressed/focus |
| `--duration-short` | 200ms — tooltip/dropdown/toast |
| `--duration-medium` | 300ms — 模态框/侧边面板 |
| `--duration-long` | 500ms — 复杂入场（**永不超过**）|

**禁忌**：bounce / spring / overshoot / 弹性曲线。

### 10.2 通用 spinner 与 CoT 显式区分

| 场景 | 实现 | 用途 |
|---|---|---|
| **通用 loading** | 弧线单段 stroke 2.5px + 800ms 匀速 | 普通加载 |
| **AI 推理中**（仅 AI 上下文）| 5 圆点 1.2s 呼吸 + 80ms 错峰 | CoT 思维链（**不可作通用 spinner**）|

### 10.3 加载状态 4 档

- < 100ms：**不显示** loading
- 100ms–1s：spinner
- 1s–10s：skeleton screen
- > 10s：progress bar + 阶段性文字

### 10.4 Toast 滞留

- success：3s
- info：5s
- warning：7s
- **error + 撤稿响应：不自动消失**（用户主动关闭）

### 10.5 错误反馈三冗余

振动（amplitude ≤ 3px）+ 红框 + 错误文字（reduce-motion 关振动保留红框+文字）

### 10.6 性能底线

- **60fps**（中端设备不掉帧）
- 仅动画 transform + opacity；**禁止** width/height/top/left
- will-change 用完即清

### 10.7 prefers-reduced-motion 兜底（全 kit 强制）

所有动效必须提供等价静态形态——功能不消失。振动 / 抖动 / 闪烁 / 循环动画完全禁用。

## 16. 错误 / 边界 / 空态文案（**03d v0.2 已签**）

### 16.1 三维分类矩阵

| 维度 | 取值 |
|---|---|
| **来源 6 类** | system / network / validation / business / auth / permission |
| **严重度 4 级** | critical → `aria-live=assertive` / error → `assertive` / warning → `polite` / info → `polite` |
| **临床敏感度 4 级**（核心特异）| **clinical-critical**（直接影响临床决策 / 处方 · 自动升 critical + 必含 PITL 提示）/ **clinical-adjacent**（影响数据展示 / 引用 · ≥ warning + 必含数据状态）/ business / cosmetic |

**4 产品默认敏感度**：DE 默认 clinical-critical/adjacent；SE 默认 clinical-adjacent/business；DI 默认 business（涉医生 PII 升 clinical-adjacent）；RCP 默认 business（涉医生反馈直接影响临床时升 clinical-adjacent）。

### 16.2 文案 4 段式（强制）

```
1. What happened（≤ 15 字 · 客观描述）
2. Why（≤ 25 字 · 不推卸不自责）
3. What to do（≤ 25 字 · 1-3 个具体行动）
4. Where to get help（仅 critical / 复杂 error · 求助路径）
```

info / cosmetic 可豁免；**clinical-critical 即使 info 也必须 4 段**。

### 16.3 错误码命名（CN/EN 双轨）

- 命名空间 5 个：`ERR-COMMON-*` / `ERR-DE-*` / `ERR-SE-*` / `ERR-DI-*` / `ERR-RCP-*`
- 编号 9 段：001-099 系统/网络 / 100-199 业务 / 200-299 验证 / 300-399 权限 / 400-499 配额 / **500-599 临床特殊** / 600-699 数据/引用 / 700-799 集成 / 900-999 内部
- **CN 用户可见 + EN 内部码双轨**（继承决议 #4）：用户可见 → CN；错误码 / 日志 / 监控 → EN
- ❌ 禁：无错误码的错误信息 / 错误码暴露技术栈（NullPointerException）/ 内部代号（如 ERR-Hippocrates-101）/ 跨产品错误码冲突

### 16.4 18 禁忌措辞

**10 通用禁忌**：未知错误 / 系统繁忙请稍后再试 / 您输入不正确 / 道歉过度 / "Oops" / 请联系管理员 / 单独 "Permission Denied" / 技术术语暴露 / 绝对化用语 / 承诺式用语（"保证修复"）/ 反问语气 / 错误中用 emoji / 全大写 "ERROR!" / 长篇道歉无 actionable

**4 临床额外禁忌**：
- "AI 替你判断 / AI 替代医生"（FTC + PITL 原则冲突）
- "肯定/必然/一定 X 病症"（监管 + 临床判断禁忌）
- 单独"建议立即停药/就诊"（医疗建议必须 PITL）
- "本系统已治愈 X 患者" 类治愈表述

### 16.5 空态 6 子类

| 子类 | 触发 | 文案核心 |
|---|---|---|
| **E1 never had data** | 首次使用 / 全新空 | 引导 + 价值钩子 + 1 个明确 CTA |
| **E2 cleared** | 用户主动清空 / 删除完后 | 简短确认 + 撤销（≤ 30 秒）|
| **E3 loading** | 数据正在拉取 | 仅 Skeleton 占位**无文字**；> 10s 才 progress bar |
| **E4 error fallback** | 加载失败 | 4 段式完整 + 重试 + 错误码 + 求助 |
| **E5 offline** | 网络断开 | 离线说明 + 重连 + 离线可用功能列表 |
| **E6 unauthorized** | 角色不够 | 当前角色 + 缺失权限 + 申请路径 |

### 16.6 临床场景特殊处理

- **PITL 核验提示触发**：confidence < 0.7 / 涉处方建议 / 涉用药剂量 / 引用文献 SourceID 失效（撤稿 / 时效到期）/ 用户标注复杂病例
- PITL 文案模板含：⚠️ + 置信度数值 + 原因 + 3 行动（医师人工核验 / 查支持证据 / **不构成处方建议**）+ L3 hotline
- **撤稿告警 UI**（继承 03f §3.4）：T+0 全屏阻塞 Toast + `aria-live="assertive"` + 不允许"忽略"或"继续使用"+ 必须刷新结论 + 触发 R1 客户告警邮件
- confidence ≥ 0.7 < 0.9 显示数值 + 证据强度 L1-L4（不阻塞）

### 16.7 求助 3 级

- **L1 自助**：FAQ + 错误码搜索（适用 info / warning / 简单 error）
- **L2 客户支持**：support@medsci-ai.com + 工单系统（适用 复杂 error / business critical）
- **L3 紧急 hotline**：+86 400-xxxx-xxxx + 临床支持专线（适用 clinical-critical / 监管告警）
- **critical + clinical-critical** 必双路径 L2 + L3
- 工单自动创建 `TASK-YYYY-NNNNN` 不含 PII

### 16.8 a11y + 动效协同（继承 04a / 04b）

- `role="alert"`（错误容器）/ `role="status"`（空态）
- critical 错误自动跳焦点；error 不自动跳（用户可控）
- **不依赖颜色**（错误必含图标 ⚠ + 文字标识）
- 触控 ≥ 44×44
- 错误反馈三冗余（振动 + 红框 + 错误文字）；reduce-motion 关振动保留红框 + 文字
- Toast 滞留：error 不自动消失 / warning 7s / info 5s
- 错误状态克制（不情绪化插画 / 不故障艺术 / 不过度装饰）

| **PATCH** | bug fix / 不改 API / 不改视觉 token / 不改 a11y 行为 | TL 单签 |
| **MINOR** | 新增组件 / 新 token / 新 props 向后兼容 / deprecation 标记 | TL + Sponsor 双签 |
| **MAJOR** | 任一 27 项 Breaking Change A1-A27 | **三签**（TL + Sponsor + 域专项）|

**v0.x.x 早期发布期**：任何变更（含 breaking）走 MINOR；**客户接入须精确锁版** `0.1.0` ✓ / 不允许 `^0.1.0` ❌。

### 18.3 27 项 Breaking Change（升 MAJOR 触发清单）

- **API 7 项**（A1-A7）：移除 export / 重命名 / props 必填项移除或语义改 / 默认值改（影响视觉）/ 事件回调签名改 / TypeScript 类型移除 / exports 字段变（导入路径变）
- **Token 3 项**（A8-A10）：删除已发布 CSS 变量 / 已发布 CSS 变量值变更 ΔE ≥ 3.0 / token 命名空间变

- **a11y 4 项**（A15-A18）：触控目标尺寸下调（44×44 → 32×32 ❌ 永不允许）/ 默认 aria 行为变 / reduce-motion 兜底逻辑改（影响降级）/ 对比度阈值降级 ❌ 永不允许

### 18.4 双向触发矩阵（13 类）

**核心原则**：品牌包文档变更**不必然**触发 NPM 版本变更；只有当文档变更**翻译为代码 / token / 资产**时才触发。

| 文档变更类型 | 触发 NPM？|
|---|---|
| voice 层修订 / 禁词清单扩充 / 架构层 02 锁定 / 04 §7 图像政策（仅文档）/ 04a a11y 规则（无对应组件改动）/ 04c 摄影插画（仅文档）| ❌ **不触发** |

### 18.5 Deprecation + Prerelease

- **Deprecation 周期**：T+0 标记 → T+1 minor 警告期 → T+2 minor 可移除（升 MAJOR）；不允许同 minor 内移除；紧急 deprecation 走 06 §2.4（24h）
- **Prerelease 4 标签**：alpha（内部尝鲜）/ beta（半公开）/ rc（最终候选）/ next（rolling）；alpha → beta 至少 1 反馈；beta → rc ≥ 1 周 + 0 critical；rc → 正式 ≥ 3 天

### 18.6 半自动发布 + 14 校验

- **CI 完成构建 + 测试 + dry-run**；最终 `npm publish` 由 TL 本地工作账号**人工触发**
- CI **绝不**做：真 publish / 自动 npm version / 自动 dist-tag latest
- **14 条校验清单**（每次 publish 前 TL 必跑）：含包大小 baseline（packed 1.0 MB / unpacked 3.3 MB · 增长警戒线 packed > 1.2 MB → 拆包评估）+ exports 9 条解析 + 主入口 39 exports + tokens 11 个 + typecheck 0 error + a11y CI pass + **§15 防 EPRIVATE**（package.json 不含 `private: true`）

### 18.7 回滚 + 客户支持

- **永不 unpublish 红线**（除合规事故 ≤ 24h）；> 24h 仅 deprecate
- 已被任何依赖方安装：不可 unpublish（即使 ≤ 24h）
- 客户支持窗口：current latest 全支持 / **n-1 minor 6 个月仅安全 patch** / **n-2 及更早不再支持** / 跳级升级强制走中间 MAJOR migration

### 18.8 依赖治理 + 多包预留 + MSH 迁移

- **依赖治理**：npm audit 0 critical / 0 high / moderate ≤ 5 跟踪；每周 cron；每月 npm outdated；Dependabot 持续；SBOM 每发版生成入实证库（与 03f §6.1 同基础设施）；保留 7 年
