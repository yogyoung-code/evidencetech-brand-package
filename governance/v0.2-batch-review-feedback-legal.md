## 4. 法务新提的 4 处修改（深度分析）

### 4.1 L1 · D5 真人医生政策修订

**原决议**（03a §6.2）：全平台禁止代言形象——含视频出镜、口播、推荐、肖像照、姓名 + 单独肖像组合。
**法务修订**：不做代言，**但可以做产品使用介绍**——可以合规。

**语义解读**：法务区分了两个概念：
- **代言（endorsement）**：医生作为"权威背书"形象出现，向公众推荐使用——合规风险高（医师代言医疗广告法律风险）
- **产品使用介绍（user-walkthrough）**：医生作为"使用者"演示如何使用产品，技术性、流程性，不构成对公众的推荐——合规可控

**接受成本**：

03a §6.2 重写（建议）：

```
真人医生政策（修订）

🚫 全平台禁止：
- 代言形象（endorsement）：医生以"权威推荐"形象向公众推荐使用产品
- 含视频代言口播 / "我推荐 / 我建议"型推荐语 / 单独肖像 + 产品 logo 组合 / "X 医生推荐"标题

✅ 允许（合规前提下）：
- 产品使用介绍（user-walkthrough）：医生作为产品使用者演示操作流程
- 必须满足：
  · 介绍内容是"我如何使用"而非"我推荐你使用"
  · 不出现"推荐 / 建议你买 / 你应该使用"等推荐语
  · 不与产品 CTA 紧邻（不能在演示后立刻接"立即购买"）
  · 医生身份披露真实（医院 + 职级 + 执业范围）
  · 演示内容限定于产品操作流（不延伸为治疗建议或处方推荐）
  · 文末附"个人使用经验，不构成医疗推荐"声明
- 仍**禁止**：
  · 任何与"医美 / 养生 / 减肥"产品的关联
  · 患者出镜（即使匿名化处理）
  · 推荐特定药品 / 治疗方案
```

**对其他文档的连锁影响**：

- 03e §1.1（含真人医生政策继承条款）需更新
- 03a §3.6 抖音 / §3.7 小红书 / §3.8 视频号——产品使用介绍允许后需在白名单加条目
- **特别关注**：R1 小红书（市场要做产品演示）+ L1（法务允许使用介绍）→ 两者兼容，**形成新的小红书内容形态：医生使用介绍 + 机制演示**（仍禁医美关联）

**风险点**：

- "使用介绍"vs"代言"的边界判定可能有灰色场景——例：医生说"我每天用 DeepEvidence 查文献"算介绍还是代言？需要法务出**判定指南** + 案例库
- 媒体监管对"医师参与广告"的审查可能不区分"代言/介绍"细分——需法务评估具体案例
- 与 03e §6.2 真人医生政策的协同（"匿名化数据 / 学术专家咨询团列表"）需联动调整

**Sponsor 建议**：法务给出"判定指南"作为 v0.2.1 patch；本批次先把"使用介绍 vs 代言"二分写入 03a §6.2。

## 5. 5 处"无需"语义澄清需求

法务对 5 处条款回复"无需"。语义有歧义——可能是：

- (a) **"无需我意见 = 直接接受"**（保留原文）
- (b) **"这条规则无需存在 = 删除"**

法务回复格式上对其他条款用"✓ ack" / "确认" / 具体修订建议；对这 5 处单独用"无需"——形成语义对比，倾向 (b) 删除，但需澄清。

| 条目 | 原决议核心 | 删除影响 | 保留理由 | 建议向法务澄清的问题 |
|---|---|---|---|---|
| **§5.5 医患纠纷"以官方新闻稿为准"统一口径** | 运营遇媒体询问统一回"以官方新闻稿为准"，转 PR 处理 | 无统一口径 → 运营各自答复风险 | 运营缺乏现场判断能力 | "统一口径是否需要在 03a 明文化？还是放在 PR 内部 SOP 即可？" |
| **D18 集采 / 一致性评价** | DI 在仿制药一致性评价 / 国家集采品种语境下不允许"帮你在医生检索中盖过原研药"暗示性表述 | 该禁忌从 03e §4.5 移除 | 集采对医生检索的影响是 DI 核心场景 | "该禁忌是否被 03e §4 政策语境 + 03f §1.2 监管 claim L4 已覆盖？还是需独立保留？" |
| **§2.4 客户 case 时效"季度"** | 客户 case 季度复审（确认仍在期内 + 客户未撤回） | 客户 case 不强制季度复审 | 客户书面同意有效期管理弱化 | "复审节奏是否过严？建议年度还是按客户合同期限？" |
| **§4.6 学术合作发布** | 联合发布的研究 marketing 化转述须经原作者 PI 书面确认；不允许 marketing 自行简化 | 学术合作转述无强约束 | PI 个人意见与营销稿差异大、合规风险高 | "PI 书面确认流程是否过重？还是放入合同条款即可，无需 03f 明文？" |
| **§9 14 条自检** | 14 条发布前自检清单 | 实证流程无 enforcement | 自检是流程闭环关键 | "14 条过细需精简？还是认为不需要在 03f 明文化（市场已有自己的清单）？" |

**Sponsor 建议**：在三方会议中**逐条澄清**——5 条都是细节而非红线，估时 10min。
