## 3. 红色冲突点深度分析

### 3.1 R1 · D7 小红书做产品功能演示

**原决议**（03a §3.7）：核心定位 = 雇主品牌 + 招聘，**不卖产品**；禁忌包含"产品功能演示"。
**理由**：受众错位 + 算法易把品牌归"消费医疗"赛道（医美 / 养生 / 减肥）+ 与真人医生政策（D5）的协同护城河。

**市场反对** = 想要做产品功能演示。

**接受成本**：
- 03a §3.7 整章重写——白名单加"产品演示"，但需新增护栏：
  - 不与医美 / 养生 / 减肥 / 体检流量话题关联（仍禁）
  - 不出现真人医生（仍禁，继承 D5）
  - 产品演示限定为：UI 操作流 / 信任徽章解释 / Diff View 演示 / Verify It 体验（**仅工程化、机制化的内容**，不是"治病推荐"）
  - 与抖音 / 视频号的内容差异化策略（避免重复推同一 demo）
- 重新评估"医美关联污染"风险——需要法务从合规角度判断
- 增加小红书算法分类的运营动作（避免被归"消费医疗"）

**Sponsor 建议立场**：折中——小红书可以做"机制 / 操作 / 信任层"演示（教育向），但仍**禁止**"功能 + 病例 / 用户故事"演示（容易激活算法）。**待法务 ack 后落地**。

### 3.2 R2 · D14 大区可以独立发布

**原决议**（03e §6.1 + §6.2）：大区**不能**独立发布对外内容；本地化稿件须经市场总部 + 法务审；大区只能提交"本地化申请单"。
**理由**：合规风险（DRG/DIP / 医保 / 互联网医院 / 数据出境跨省差异；任何错位文案都有 Legal 风险）+ 案例本地化跨层禁忌（避免大区拿三甲 case 推县域）。

**市场反对** = 想给大区独立发布权。

**接受成本**：
- 03e §6.1 RACI 重写——T3/T4 落地内容审签从"市场总部+法务"降为"大区+总部知会"
- 必须配套：
  - **大区 SOP**：大区可独立发布的内容范围（vs 必须经总部的内容）；建议分二级——A 类完全自由（活动通告 / 学术合影 / 团队动态）/ B 类可发但 24h 内总部抽查（产品演示 / 客户案例 / 政策评论）
  - **大区培训认证**：每位大区市场对接人通过 voice 培训认证
  - **抽查机制**：总部每周抽查 N% 已发布内容，触发违规则升级处罚
  - **出错连带责任**：违规帖文涉及合规事件，大区直接负责人 + 总部市场负责人共同担责
  - **撤回兜底**：大区独立发布的内容如出错，撤回时间窗按 03a §8.2（公众号 2h / 微博 12h / 等）
- 与 03f §3.4 撤稿响应分类处理的兼容——大区独发的 case 引用如撤稿，反向追踪难度增加

**Sponsor 建议立场**：折中——**A 类大区独立发布**（活动 / 团队动态 / 学术合影 / 不带数据 claim 的常规内容），**B 类仍走总部审**（产品演示 / 客户案例 / 政策评论 / 含数字的 claim / 跨省政策语境）。**法务必签**——这条直接影响合规边界。

### 3.3 R3 · D20 SourceID 体系可否省略

**原决议**（03f §2.1 + §2.2）：每条引用源进入对外物料前必须先在实证库注册获 SourceID；状态 approved 才可引用；不允许静默修改。
**理由**：撤稿响应（§3）依赖反向追踪；跨场景一致性（§5.1）依赖单一源；审计留痕；监管 / 投资者关系材料的合规要求。

**市场反对** = 想省略（**未明确程度**——可能是"全部省略"也可能是"轻量化"）。

**接受成本（全省略）**：
- 03f 整个文档崩塌——§2 / §3 / §4 / §6 / §7 / §9 全部依赖 SourceID
- 撤稿响应能力丧失——文献撤稿时无法反向追踪受影响 claim
- 监管 / 投资者关系材料合规风险显著上升
- 客户合同条款承诺无 audit trail
- **强烈不建议**

**接受成本（轻量化）**：
- L3 / L4 强制 SourceID（同行评议研究 / 官方文件 / 客户书面同意）；L1 / L2 可松（公开来源 / 内部审计可用简化标注）
- 对应改动：
  - §2.1 字段强制清单 → 区分必填字段 vs 选填字段（按 L 级）
  - §3 撤稿响应 → L3 / L4 标准流程；L1 / L2 简化版
  - §6.1 实证库工具 → 轻量版可用 Notion 标签替代完整数据库
  - §9 自检 → 14 条减为 10 条

**Sponsor 建议立场**：**澄清反馈**是"全省略"还是"轻量化"。如全省略 → 法务大概率反对，预计在三方会议中反弹；如轻量化 → 可接受，按 L 级分级强制。**先问清楚**。

### 3.4 R4 · D23 产品 UI claim 不用走 RFC + 法务

**原决议**（03f §4.3）：UI 上的性能 / 准确率 / 覆盖范围 / 案例计数等数字全部有 SourceID；claim 修改走产品 RFC + 法务审；A/B 测试每个变体须独立 SourceID。
**理由**：UI claim 是离用户最近的 voice 触点；与营销材料相同的合规风险；A/B 实验如不留痕等同于"用户面前做未审实验"。

**市场反对** = UI claim 不走 RFC + 法务。

**接受成本**：
- 03f §4.3 重写——UI claim 改动允许工程 sprint 内自由修改
- 风险：
  - UI 上"覆盖 N 家三甲"等数字脱审；如客户 / 监管质疑无 audit trail
  - 撤稿事件 → UI 改动不留痕 → 影响范围分析困难
  - 营销 / 销售 / 投资者 / UI 表述不一致（违反 §5.1 跨场景一致）
- **法务大概率不接受这条**（与监管语境冲突）

**Sponsor 建议立场**：**分级**——
- **静态 UI claim**（"覆盖 X 家医院" / "n=Y, 来源" / "准确率 91.3%"）必走 RFC + 法务（保留原决议）
- **动态 dashboard 数字**（"今日 X 单" / "本周新增 Y 位医生"）= 后台日志直读 → 不需 RFC + 法务，但**必须配置上限报警**（异常值如负数、超过历史 10 倍则不显示）
- **A/B 测试变体**仍需 SourceID（继承原决议）
- **临床效果 UI claim**（"降低误诊率 X%"）必走 RFC + 法务 + 学术（最高级别）

提议折中：D23 不"全推翻"也不"全保留"——按 UI claim 类型分级。**法务必裁**。

### 3.5 R5 · §8 受众分层适配 vs §1.2 实证矩阵 矛盾

**反馈原文**：§1.2 实证矩阵 = 要求；§8 受众分层适配 = 不用。

**矛盾分析**：

03f §8 的"受众分层适配"原文（§5.2）：

```
T1 / T2 / MNC → L3 SCI + L2 内部审计
T3 → L3 SCI + L4 国家指南
T4 → L4 国家指南 + L3 中文综述
T5 → L4 客户 case（书面同意）+ L2 业务结果
```

这是 §1.2 实证矩阵在 03e 受众分层下的具体应用。如果接受 §1.2 但不接受 §8——即**所有受众层级统一用 §1.2 矩阵**，不区分 T1 vs T4 的引用源偏好。

**接受成本**：
- 实证统一→简化（无需按受众层做引用源类型适配）
- 但与 03e §2.2 引用密度分层冲突——03e 已说"T1 偏 SCI、T4 偏国家指南"
- 需要二选一：
  - 选项 A：保留 03e §2.2 + 03f §1.2 + 删除 03f §5.2 的受众分层适配 → 03e 主导引用源选择，03f 仅管最低实证级别
  - 选项 B：保留 03f §5.2 + 03e §2.2 → 两套规则统一表达
  - 选项 C：删 03e §2.2 + 03f §5.2，所有受众都用 §1.2 矩阵 → 简化但放弃分层精细度

**Sponsor 建议立场**：选项 A 最符合市场反馈意图——"§1.2 矩阵管最低实证级别（合规底线）；引用源类型偏好回到 03e 处理（分层精细度）"。这个解释最合理。**待澄清确认**。

## 7. 提议下一步（Sponsor 决策点）

按 packet §8 反馈与退回流程，Sponsor 三选项：

### 选项 A · 等齐法务再开三方会（保守）

- 当前路径：等法务 review 完成 → 综合两方反馈 → 一次性召开 Sponsor + 市场 + 法务三方会议
- 优势：议题完整、一次裁决；避免市场 / 法务交替修改
- 劣势：法务 review 估 ~3h，加上协调可能 5–7 个工作日；总周期拉长
- 适用：法务非常严格、预计大量 push back

### 选项 B · 立即开三方会（折中）⭐ 建议

- 当前路径：现在召集 30min 三方会议（Sponsor + 市场 + 法务），把市场已反馈的 5 个红色冲突点 + 4 个黄色配套项 + 选项澄清同时讨论；法务在会上做"快速判断"对市场反馈的 5 条直接表态，避免来回
- 优势：快速收敛；市场 / 法务在场对质比来回邮件高效；Sponsor 可以现场拍板
- 劣势：法务 30min 内能否充分判断 R3 / R4 这种深度议题存疑；可能仍需 follow-up
- **适用：当前情况**——5 红 + 4 黄足够触发会议；法务对其他条款（D5 / D8 等）可在会后补 review

### 选项 C · Sponsor 单方裁决（激进）

- 当前路径：Sponsor (Yog) 直接拍板修订源文档 → 跳过三方会议 → 重发第二轮签
- 优势：最快；保留 Sponsor 决策权
- 劣势：违反 06 §2.1 双签流程；市场 / 法务 ack 流程被绕开；后续追究合规事件时责任不清
- **不推荐**
