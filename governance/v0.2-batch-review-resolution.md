## 3. 真正需 Sponsor 三方会议拍板的 2 红点

### 3.1 R2 · 大区独发权

**冲突状态**：

- 市场：大区**可以**独立发布
- 法务：✓ ack 03e §6.1 RACI（即接受"大区**不能**独立发布"原 RACI）
- **真正冲突**

**Sponsor 建议折中方案**（优先讨论）：

| 内容类型 | 大区权限 | 总部权限 |
|---|---|---|
| **A 类** 活动通告 / 团队动态 / 学术合影 / 不带数据 claim 的常规内容 | **大区独发** | 总部 24h 内抽查 N% |
| **B 类** 产品演示 / 客户案例 / 政策评论 / 含数字 claim / 跨省政策语境 / 政府对接 | 大区起草 | **总部 + 法务必审** |

**讨论焦点**：

1. A/B 边界判定——含"数据 claim"边界（"我们参与了 X 会"算不算？）；建议法务给"判定指南"案例库
2. 抽查机制——抽查频次 / 违规处罚 / 处罚级联（大区责任 + 总部连带）
3. 大区培训认证 / 撤回时间窗（按 03a §8.2 收紧版）/ 出错连带责任
4. 与 03f §3.4 撤稿响应分类处理的兼容——大区独发的 case 引用如撤稿，反向追踪难度

**会议要决议**：

- [ ] 接受 A/B 类拆分（如否，议题升级到 Sponsor 单方裁决）
- [ ] A 类边界由谁判定（市场总部？法务？或工具化的"判定 wizard"）
- [ ] 抽查频次 / 处罚机制
- [ ] 法务出"判定指南" timeline

**预计时长**：15min

### 3.2 R3 · SourceID 省略 vs 轻量化

**冲突状态**：

- 市场：可否**省略**（语义未明：全省略 / 轻量化）
- 法务：✓ ack D20（明确接受 SourceID 不允许静默修改）+ ✓ ack D21 / D22 整套体系
- **真正冲突 — 法务坚决反对全省略**

**Sponsor 建议**：

**第一步澄清市场反馈语义**：

- 选项 X：**全省略**——03f 整章崩塌，**法务大概率坚决反对**，会议中可能升级
- 选项 Y：**轻量化**——按 L 级分必填/选填字段，L3/L4 强制 SourceID，L1/L2 简化版

**预测市场实际意图是 Y（轻量化）**——因为：

1. 市场未反对 §1.2（实证矩阵）+ D24（合同 L4 三签）+ D22（撤稿响应铁律）
2. 单方反对 D20 意味着推翻 03f 整章，但市场对其他实证条款都接受
3. 反馈用"可否"而非"删除"——更接近询问能否轻量

**轻量化方案**（讨论起点）：

| 实证级别 | SourceID 强制度 | 字段最小集 |
|---|---|---|
| **L1 公开来源** | 简化标注（行业报告标题+年份+作者+URL）| 5 个核心字段 |
| **L2 内部审计** | 标准 SourceID（带文件副本）| 8 个字段 |
| **L3 同行评议** | 完整 SourceID（强制 PMID/DOI + 审批 trail + 反向链接）| 12 个字段（原版）|
| **L4 官方文件** | 完整 SourceID + 监管文号关联 | 14 个字段（含监管字段）|

**会议要决议**：

- [ ] 澄清市场反馈：全省略 vs 轻量化
- [ ] 如轻量化，接受按 L 级分级方案
- [ ] L1/L2 简化版的具体字段集
- [ ] 与 §3 撤稿响应的兼容（L1/L2 反向追踪能否依赖标签替代字段）
- [ ] 与 §4.3 UI claim 的协同（连带影响 R4）

**预计时长**：15min

### 3.3 R4 · UI claim 不走 RFC + 法务（隐含冲突，关联 R3）

**冲突状态**：

- 市场：UI claim **不走** RFC + 法务
- 法务：未明确表态 D23，但 ✓ ack D24 / D20 / §1.2 → 隐含坚持原决议
- **法务态度需现场确认**

**Sponsor 建议折中方案**（与 R3 协同讨论）：

| UI claim 类型 | 流程 |
|---|---|
| **静态 claim**（"覆盖 X 家医院" / "n=Y, 来源" / "准确率 91.3%"）| 走 RFC + 法务（保留 D23 原决议）|
| **动态 dashboard 数字**（"今日 X 单" / "本周新增 Y 位医生"）| 后台日志直读 → 不走 RFC + 法务，但**必须配上限报警**（异常值不显示）|
| **A/B 测试变体** | 仍需 SourceID（原决议）|
| **临床效果 UI claim**（"降低误诊率 X%"）| 三签（产品+学术+法务）—— 最高级别 |

**会议要决议**：

- [ ] 法务对此分级方案的接受度
- [ ] 静态 vs 动态 claim 的边界判定（需法务给指南）
- [ ] 上限报警的工程实现（需产品 / 工程后续 follow-up）

**预计时长**：10min
