### 4.1 错误分类（4 条 · D1–D4）

| # | 决议 | 出处 |
|---|---|---|
| **D1** | **错误分类三维矩阵**：来源 6 类（system / network / validation / business / auth / permission）× 严重度 4 级（critical / error / warning / info）× **临床敏感度 4 级**（clinical-critical / clinical-adjacent / business / cosmetic）—— 后者为本子文档核心特异维度 | 03d §1 |
| **D2** | 严重度对应 aria-live：critical / error → `assertive`；warning / info → `polite`（继承 04a §4）| 03d §1.2 |
| **D3** | **临床敏感度 4 级定义**：clinical-critical（直接影响临床决策 / 处方）严重度自动升 critical + 必含 PITL 提示；clinical-adjacent（影响数据展示 / 引用）建议 ≥ warning + 必含数据状态；business 标准；cosmetic 后台修复不显式报错 | 03d §1.3 |
| **D4** | **4 产品默认敏感度**：DE 默认 clinical-critical/adjacent；SE 默认 clinical-adjacent/business；DI 默认 business（涉客户医生 PII 升 clinical-adjacent）；RCP 默认 business（涉医生反馈直接影响临床决策时升 clinical-adjacent）| 03d §1.4 |

### 4.2 文案 4 段式 + 错误码命名（4 条 · D5–D8）

| # | 决议 | 出处 |
|---|---|---|
| **D5** | **文案 4 段式强制**：What happened（≤ 15 字）/ Why（≤ 25 字 不推卸不自责）/ What to do（≤ 25 字 1-3 个具体行动）/ Where to get help（仅 critical / 复杂 error）；info / cosmetic 可豁免；clinical-critical 即使 info 也必须 4 段 | 03d §2 |
| **D6** | **错误码命名空间 5 个**：ERR-COMMON-* / ERR-DE-* / ERR-SE-* / ERR-DI-* / ERR-RCP-*（与 04 批次 BrandScope 协同）| 03d §3.1 |
| **D7** | **错误码编号 9 段**：001-099 系统/网络 / 100-199 业务 / 200-299 验证 / 300-399 权限 / 400-499 配额 / 500-599 临床特殊 / 600-699 数据/引用 / 700-799 集成 / 900-999 内部 | 03d §3.2 |
| **D8** | **CN 用户可见 + EN 内部码双轨**（继承决议 #4）：用户可见 → CN；错误码 / 日志 / 监控 → EN；EN 用户可见仅 EN 单语场景；每个错误码必有 EN 内部码 + CN 用户可见 + EN 用户可见对照 1:1 等价 | 03d §3.3 + §11 |

### 4.3 18 禁忌措辞（3 条 · D9–D11）

| # | 决议 | 出处 |
|---|---|---|
| **D9** | **10 通用禁忌**：未知错误 / 系统繁忙请稍后再试 / 您输入不正确 / 道歉过度 / Oops / 请联系管理员 / 单独 Permission Denied / 技术术语暴露 / 绝对化用语 / 承诺式用语 / 反问语气 / emoji（错误中）/ 全大写 / 长篇道歉无 actionable | 03d §4.1 |
| **D10** | **4 临床额外禁忌**："AI 替你判断 / AI 替代医生" / "肯定/必然/一定 X 病症" / 单独"建议立即停药/就诊"（医疗建议必须 PITL）/ "本系统已治愈 X 患者" 类治愈表述 | 03d §4.2 |
| **D11** | 错误码不允许形态：无错误码的错误信息 / 错误码暴露技术栈 / 错误码使用产品内部代号（如 ERR-Hippocrates-101）/ 跨产品错误码冲突（每个码全局唯一）| 03d §3.4 |

| # | 决议 | 出处 |

### 4.5 临床场景特殊处理（4 条 · D13–D16）
