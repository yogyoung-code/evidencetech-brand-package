## 0. 适用范围与边界

### 0.1 覆盖

本文档覆盖梅斯 AI / MedSci AI 子品牌**所有**涉同行 / 竞品 / 对比的表述：

- **公开渠道**：官网 / 落地页 / 白皮书 / 社交媒体 / 媒体采访 / 学术合作 / 招股材料 / 季报年报 / 客户合同
- **内部渠道**：Battle Card / 销售脚本 / 客户支持回复模板 / 内部培训材料 / 季度市场分析报告
- **半公开**：客户提案 / RFP 响应 / 投标书 / 客户共发布材料

### 0.2 不覆盖

- **学术发表**——按 03f §4.6 学术合作合同模板化处理（员工以个人身份发表的论文中引用同行研究是科研学术行为，按学术规范）
- **员工个人言论**——员工以个人身份在 LinkedIn / 朋友圈 / 知乎对同行的观点；按 03 + 03a 处理
- **法律诉讼 / 监管举报**——专门法律流程；不在本文档范围
- **公开股票市场分析**——投资者关系层面的同行对比按 HKEX 信披规则（继承 03f §3.4 招股材料 HKEX 路径）
- **兄弟业务（梅斯医生）非 AI 子品牌内容**——按母品牌 v1.1 + 03 §5.4 严格不互换原则

### 0.3 优先级层级

冲突时按以下顺序裁决：

1. 03f §3.4 撤稿响应铁律（涉同行论文被撤稿时）
2. 03 §6 禁词清单（绝对化用语 / 监管禁忌）
3. 03 §5.4 兄弟业务严格不互换
4. 03a §5.4 社交场景同行默认沉默
5. 06 §2.4 紧急修订流程（监管事件触发）
6. 本子文档（竞品对比层附加约束）

## 1. 同行分类

### 1.1 三类同行 / 竞品

| 类别 | 描述 | 示例 | 在本文档中的处理 |
|---|---|---|---|
| **C1 直接竞品** | 同领域 AI 医疗 SaaS / Decision Support / 临床智能 | Watson Health / 国内某医疗 AI 厂商 / 某临床决策辅助产品 | 完全适用本文档全部规则 |
| **C2 间接竞品** | 传统 CDS / 医疗信息系统 / 通用医学搜索引擎 | Wolters Kluwer UpToDate / Lexicomp / Micromedex / 某医学搜索 | 按本文档规则处理；公开声明可提"产品类别"但不点名 |
| **C3 兄弟业务（梅斯生态内部）** | 梅斯健康集团内部其他业务平台（医生社区 / IIT 平台 / 文献中心）| 梅斯医生（医生社区）/ MedSci Online | **严格不互换**（继承 03 §5.4）：AI 子品牌物料中不能假称是梅斯医生提供 / 不能挪用梅斯医生 case 给梅斯 AI；可在公开渠道**正向提及**作为协同生态 |
| **C4 上下游伙伴 / 合作方** | 与梅斯 AI 集成 / 协同的非竞品方 | EHR 厂商 / 医学影像厂商 / 某药企作为客户 | **不属于竞品**；按合作伙伴规则处理（参 03c §1.4 客户共发布）|

### 1.2 关键鉴别问题

判断"是否构成竞品对比"的 4 个核心问题：

1. ✓ 是否在我方物料中**出现同行品牌名 / 产品名**？（即使首字母也算）
2. ✓ 是否暗示"我方更好 / 同行更差"？
3. ✓ 是否引用同行**未公开**的内部数据 / 内部材料？
4. ✓ 是否会被同行 / 媒体 / 监管解读为**贬损 / 虚假宣传 / 不正当竞争**？

任一答 ✓ 即触发本文档规则。

### 1.3 与 03 §5.4 兄弟业务规则的关系

兄弟业务（梅斯医生）**不是竞品**，但因品牌混淆风险（用户可能以为"梅斯医生 = 梅斯 AI"），需严格不互换：

- ✓ 公开渠道可提及"梅斯生态包括 AI 子品牌 + 医生社区 + IIT 平台"（正向协同）
- ❌ 不能在 AI 子品牌物料中**挪用**梅斯医生的医生网络数据 / case 作为 AI 产品的能力宣传
- ❌ 不能让客户误以为购买梅斯 AI 自动包含梅斯医生服务

## 2. 5 条公开声明铁律

### 2.1 R1 — 不点名

公开渠道**永远不直接提及**竞品名称：

- ❌ "梅斯 AI 比 [Watson Health] 准确率高 X%"
- ❌ "我们使用与 [某品牌] 相同的算法但精度更高"
- ❌ "替代你正在使用的 [某产品名]"
- ✓ "在临床决策辅助类产品中，DeepEvidence 在 [具体场景] 的准确率达 91.3%¹"（不点名）
- ✓ "针对当前业内常见的 X 痛点，我们提供 Y 解决方案"

**例外**：

- 学术合作论文中按学术规范引用同行研究（学术发表 · 不在本文档范围）
- HKEX 上市公司披露的法定要求（如招股书中竞争格局分析 · 法务详读 + Sponsor 拍板）

### 2.2 R2 — 不暗示业内最强 / 唯一

避免任何**暗示**自己是最强 / 唯一 / 第一的表述（同时违反 03 §6 绝对化禁忌）：

- ❌ "业内唯一通过 PITL 全验证的 AI 临床决策"
- ❌ "市场上准确率最高的医疗 AI"
- ❌ "全行业首个支持 X 的产品"
- ❌ "100% 准确" / "永远不出错"
- ✓ "DeepEvidence 在 [n=1,234 案例 · 2025] 测试中达到 91.3% 准确率¹"（具体数字 + SourceID）
- ✓ "支持 PITL 全流程验证（参 §X 视觉系统）"

监管层面这类暗示性表述可能触发：

- NMPA 医疗器械广告法
- FTC 比较广告法（出海场景）
- 反不正当竞争法
- 虚假宣传相关条款

### 2.3 R3 — 不引用同行内部材料（继承 03f §2.3）

不允许引用作为对比依据的来源：

- ❌ 同行的内部测试数据 / 客户邮件 / 演示视频
- ❌ 同行未发布的功能 / 未公开的 roadmap
- ❌ 同行客户的"私下吐槽"
- ❌ 已撤稿的同行论文（继承 03f §2.3）
- ❌ 同行未授权使用的 case study
- ❌ 维基百科 / 百度百科条目（继承 03f §2.3）
- ❌ 第三方未审计的对比报告（如某博主 / 某分析师"我用过 N 个产品"）

**允许**作为对比依据的来源（必须实证库 SourceID 注册）：

- ✓ 同行**公开发布**的产品文档 / 官方白皮书 / 上市公司公告（HKEX / SEC）
- ✓ 同行**正式发表**的同行评议论文（L3 实证强度）
- ✓ NMPA / FDA / EMA 公开的产品认证 / 召回信息
- ✓ 公开行业报告（IDC / Gartner / Frost & Sullivan · L2 实证）

### 2.4 R4 — 不挑起监管 / 法律纠纷

任何对比表述都要避免：

- ❌ 暗示同行违规 / 违法 / 被监管处罚
- ❌ 引用监管处罚信息作为竞品贬损
- ❌ 在媒体上"披露"同行内部消息
- ❌ 主动联系监管举报同行（如有合规问题，按 06 §2.4 内部流程 + 法务）
- ✓ 监管处罚信息仅供内部 Battle Card 参考（公开可查 + SourceID 注册）+ 不主动传播

### 2.5 R5 — 不在公开渠道做 vs 对比

公开渠道（社交 / 官网 / 落地页 / 白皮书 / 媒体）**绝对不允许**：

- ❌ "梅斯 AI vs [某品牌]" 对比表
- ❌ "我们 vs 他们" 功能对比矩阵
- ❌ 任何形式含两个或更多品牌名的并列对比

**允许**：

- ✓ 聚焦自己产品的纯描述（"DeepEvidence 提供 X / Y / Z 能力"）
- ✓ 提供"如何评估 AI 临床决策辅助产品"评估框架（不点名 / 客户自己对比）
- ✓ 公开可查的行业格局分析（如"医疗 AI 市场分类"）
- ✓ HKEX 上市公司披露强制要求的竞争格局段落（按法务 + Sponsor 拍板逐字）

## 3. 内部 Battle Card 规范

### 3.1 用途与限定

| 维度 | 规范 |
|---|---|
| **使用者** | 仅 BD / 销售 / 客户支持 / 高级管理层（白名单制度）|
| **使用场景** | 客户对话准备 / 销售脚本参考 / RFP 响应背景知识 |
| **不允许场景** | 不外发客户 / 不打印 / 不发邮件附件 / 不在公开演示中展示 / 不在销售 LinkedIn 个人账户分享 |
| **存储位置** | 内部 Wiki / 加密 SaaS（与实证库工具 03f §6.1 同基础设施 · 待 v0.2 决议会议确定）|
| **访问权限** | 单点登录 + 角色权限 + 访问日志（每次查看记录留存 7 年）|
| **更新频次** | 每季度 review；同行重大变更（产品发布 / 监管事件 / 收购）触发临时更新 |

### 3.2 Battle Card 字段

每个直接竞品（C1）一份 Battle Card；统一字段：

```
1. 同行品牌名 + 产品名
2. 同行所在公司（含上市状态 / 母公司）
3. 同行核心定位（基于其官方文档）
4. 同行产品功能（按公开材料列出）
5. 同行客户类型 / 主要场景
6. 同行公开数据 claim（必带 SourceID · 实证库注册）
7. 同行公开监管资质（NMPA / FDA / CE 等）
8. 同行近期动态（产品发布 / 融资 / 收购 / 监管事件）
9. **梅斯 AI 在每个维度上的对应能力**（不褒贬，仅事实）
10. 客户对话建议 talking points（应答框架，参 §4）
11. 上次更新日期 + 更新人 + 法务 review 状态
```

### 3.3 Battle Card 严禁的字段

| ❌ 严禁字段 | 原因 |
|---|---|
| **同行的"弱点 / 痛点 / 缺陷"列表** | 易被误用为公开诋毁素材 |
| **同行客户的"差评 / 抱怨"** | 信息源不可靠 + 法律风险 |
| **"如何挖角同行客户"剧本** | 不正当竞争 |
| **同行内部消息 / 内部邮件 / 离职员工口述** | 法律风险（商业秘密 + 不当获取）|
| **同行未公开 roadmap / 未发布功能预测** | 信息源问题 |
| **个人评价（"X 公司 CEO 不靠谱"）** | 诽谤风险 |

### 3.4 Battle Card 信息源原则（继承 03f §2.3）

每个字段必须有 SourceID 注册（实证库）；信息源仅限：

- ✓ 同行官网 / 官方白皮书
- ✓ 同行 SEC / HKEX / 港交所披露
- ✓ 同行同行评议论文（L3）
- ✓ NMPA / FDA / EMA 公开数据库
- ✓ 公开行业报告（IDC / Gartner / Frost & Sullivan）
- ✓ 公开新闻报道（主流媒体 · 不含博客 / 自媒体 / 谣传）

### 3.5 Battle Card 季度审计

每季度由市场 + 法务联合审计：

- 抽样 ≥ 50% Battle Cards
- 检查信息源是否仍可访问 / SourceID 是否仍 approved
- 检查公开数据是否过时（> 12 个月需复审）
- 检查是否有任何字段违反 §3.3
- 法务 review 状态更新

## 4. 客户问"你们 vs X"应答框架

### 4.1 4 步应答原则（销售 / 客户支持脚本基础）

```
Step 1 — 不诋毁
不评价同行优劣；不暗示同行有问题。

Step 2 — 客户驱动
"你为什么对比这两个产品？" / "你最关心 [场景 X] 的什么维度？"
→ 让客户说出真实需求

Step 3 — 聚焦自己
基于客户说出的需求，介绍梅斯 AI 在 [具体场景] 的能力（带数据 + SourceID）

Step 4 — 鼓励客户自己评估
提供"如何评估 AI 临床决策辅助产品"评估框架（不含品牌名）；
建议客户做小规模 PoC / 试用 / 同行评议
```

### 4.2 应答模板示例

#### 客户问："你们和 [Watson Health] 比怎么样？"

应答（按 4 步原则）：

```
[Step 1 — 不诋毁]
都是临床决策辅助产品；很多客户在评估期会比较多家。

[Step 2 — 客户驱动]
能问下你最关心哪几个维度吗？
比如：(1) 中文支持 (2) 与你们医院 EHR 集成的方式 (3) PITL 验证流程 (4) 数据合规

[Step 3 — 聚焦自己 · 基于客户回答]
（如客户答中文支持）
"梅斯 AI 是中文为主语言开发的；DE 在 [n=1,234 临床场景测试 · 2025] 准确率达 91.3%¹。
我们的 PITL 验证流程（医师在回路）可以让每条结论都可验证 / 可溯源。"

[Step 4 — 鼓励客户自己评估]
"你可以先在你们 [科室] 做 30 天 PoC 试用；我们提供 [N] 个临床场景的真实数据，
你们医生可以亲自验证。也建议你看 [评估框架链接]，
里面列出了评估临床 AI 产品的 12 个维度。"

【数据出处】
¹ EVID-2026-0042 · 内部对照 · 2026-04 报告
```

### 4.3 不允许的应答

- ❌ "Watson Health 在中文场景表现一般"（评价同行 · 违反 R1 R2）
- ❌ "他们 [指 Watson] 的算法和我们差不多但更慢"（贬损 + 内部信息推测）
- ❌ "[某品牌] 最近被监管警告了"（即使是事实也不在客户对话中讲）
- ❌ "他们的客户后来都迁到了我们这"（即使是事实也是不正当竞争表述）

### 4.4 客户继续问追问时

如客户坚持"你必须告诉我和 X 比谁更好"：

```
"我理解你想做出最好的选择。但 (1) 不同产品在不同场景下各有所长，
单维度的'更好'不一定符合你的真实需求；(2) 我们的政策是聚焦自己的能力描述，
不评价同行。我建议你直接看两边产品的真实演示，让医生团队亲自评估。"
```

## 5. 媒体记者问对比的应答（协同 03c §M8）

### 5.1 媒体场景的特殊风险

媒体场景下任何对竞品的表述：

- 都会被引用 / 截屏 / 二次传播
- 都会被竞品法务关注
- 都可能被解读为"行业内幕"
- 都可能引发监管关注

→ 媒体场景**比客户对话更严格**。

### 5.2 媒体应答模板（基于 03c §M8 媒体采访模板增强）

#### 记者问："你们怎么看 [某品牌] 最近发布的 X 功能？"

应答：

```
[1 — 礼貌不评价]
"感谢你的问题。我们一向不评价同行的具体产品。"

[2 — 转向行业层面]
"如果就行业现象来说，临床 AI 产品的 X 趋势（不点名）反映了
当前医生对 [具体痛点] 的真实需求。这也是我们在 [产品名] 重点投入的方向。"

[3 — 聚焦自己 · 数据 + SourceID]
"以梅斯 AI 旗下 DeepEvidence 为例，我们在 [n=1,234] 临床场景测试中
准确率达 91.3%¹。我们采用 PITL（医师在回路）原则——
每条 AI 结论可验证 / 可溯源。"

[4 — 必要时提供资料]
"如果你想深入了解我们的产品，我可以提供 [产品文档链接 + 客户案例]。"

【数据出处】
¹ EVID-2026-0042 · 公开链接：[链接]
```

### 5.3 媒体场景禁忌

继承 §2 R1-R5 全部铁律 + 媒体特异：

- ❌ 不接受"off-the-record" 评论同行（都会被记下来）
- ❌ 不在记者面前比较产品截屏
- ❌ 不提供同行客户名单（即使你知道）
- ❌ 不主动提及同行近期动态（让记者自己说，你只回应）

### 5.4 媒体场景升级流程

任何媒体涉同行问题，记者一旦提出：

1. **当场不深入**——按 §5.2 应答框架礼貌带过
2. **24h 内通知 PR + 法务**——汇报采访内容（即使没问题）
3. **PR + 法务 review**——评估是否需要主动联系记者澄清 / 修订

## 6. RFP / 投标响应中的对比处理

### 6.1 客户在 RFP 中要求"vs 竞品对比表"

常见情况：客户的 RFP 文档中会要求供应商填写"与竞品对比"表格。

应对策略（**法务 + 市场双签每次响应**）：

```
策略 A — 重新框定
"我们一向不与同行做并列对比；可提供针对你 RFP 中的 [具体维度]
我们产品的能力描述 + 数据 claim + SourceID + 客户案例。"

策略 B — 提供评估框架
"附上'医疗 AI 临床决策辅助产品评估框架 v1.2'（不含具体品牌名）;
可作为你们评估 RFP 各供应商的统一标准。"

策略 C — 直接拒绝并说明
"基于公司合规政策，我们不能在投标材料中直接对比同行产品。
你可以参考我们 [n] 个客户的真实使用反馈做评估。"
```

### 6.2 投标响应中允许的"对比"形式

- ✓ "梅斯 AI 在 [12 个评估维度] 的能力描述 + 数据 + SourceID"（聚焦自己）
- ✓ "我们采用 PITL 原则 / Verified Generative Intelligence™ 技术 / Nano-Citation 引用机制"（独有 / 差异化卖点）
- ✓ 提供评估框架供客户对比

### 6.3 投标响应中禁忌

- ❌ 含同行品牌名的对比表
- ❌ 暗示同行劣势（即使隐晦）
- ❌ 引用同行内部材料 / 离职员工口述
- ❌ 在投标演示中现场对比同行截图

### 6.4 投标响应签字流程

每个涉及同行表述的 RFP 投标响应**必须**：

1. 法务详读最终版（≥ 24h 提前提交）
2. 市场负责人 + 法务双签（书面）
3. 大区销售独发**不允许**（继承 03e §6.1 例外条款）
4. 跨产品 / 客户 ARR ≥ 100 万 → Sponsor 信息周知

## 7. 学术合作中引用同行研究

### 7.1 学术合作论文（与梅斯 AI 联合署名）

学术合作论文中按学术规范引用同行研究：

- ✓ 在文献综述部分客观引用同行公开论文（L3 同行评议）
- ✓ 引用同行论文的实验数据 / 方法（必须 cite + 不歪曲）
- ✓ 在 Discussion 部分对比方法论 / 结果（学术规范允许）
- ✓ 同行论文 SourceID 注册（继承 03f §2）

不允许：

- ❌ 在论文中暗示同行方法有缺陷而我们更好
- ❌ 引用同行未公开数据 / 内部材料
- ❌ 论文标题或摘要含"vs [某品牌]" 的对比

### 7.2 学术合作论文签字（继承 03f §4.6）

学术合作论文涉同行引用必须法务 + 临床顾问 + 论文 PI 三方签：

- 法务：信息源合规 + 不构成虚假宣传
- 临床顾问：方法论 + 临床合理性
- 论文 PI：学术诚信

### 7.3 内部学术分享（员工内部学习）

员工内部分享某同行论文（学习 / 培训）：

- ✓ 完整 cite 同行论文
- ✓ 客观陈述方法论
- ❌ 不在内部分享 deck 中加 "vs 我们" 对比（即使是内部用 · 防止 leak）

## 8. 同行撤稿 / 监管事件中的表述

### 8.1 同行论文被撤稿（协同 03f §3.4）

继承 03a §5.1 + 03f §3.4 规则：

- 我方在公开渠道**保持沉默**——不借机攻击 / 不发"我们和他们不一样"内容
- 仅在内部 Battle Card 字段更新（同行近期动态 / 撤稿事件 + SourceID）
- 如客户主动问"X 论文被撤稿对你们影响"：
  - 应答：聚焦自己（"我们的 PITL 验证流程是 [描述]"）+ 不评价同行
- 如撤稿涉及与梅斯 AI **同领域**研究 → PR + 法务 review 是否需主动声明（区分自己使用的研究是否受影响）

### 8.2 同行被监管警告 / 处罚

- 公开渠道**保持沉默**——不发"我们合规他们违规"
- 内部 Battle Card 更新（同行近期动态字段 · 监管处罚事件 + SourceID 链接到 NMPA/FDA 公告）
- 不向监管举报（合规问题走 06 §2.4 内部流程）
- 如客户问"X 公司被监管警告"对梅斯 AI 影响：
  - 应答：中性陈述事实（"我们看到了相关公告"）+ 强调自己合规（"我们 [具体合规措施]"）+ 不评价对方

### 8.3 同行客户"投诉" / 网络负评

- 即使在网络上看到大量同行客户负评：
  - **不利用**作为对比素材
  - **不在 Battle Card 中**记录客户负评（信息源不可靠 + 法律风险）
  - **不向自己客户引用**同行客户负评

### 8.4 同行被收购 / 退市 / 倒闭

- 公开渠道**沉默**
- 内部 Battle Card 更新动态字段
- **不主动**联系同行客户挖单（继承 §3.3 严禁字段）
- 如同行客户主动来询：按正常销售流程；不暗示"你的旧供应商不行了"

## 9. 跨境与多区域差异

### 9.1 中国大陆（NMPA 监管 · 主战场）

- 适用反不正当竞争法 + 广告法 + 医疗器械广告管理办法
- §2 R1-R5 公开声明铁律**严格执行**
- 内部 Battle Card 季度法务审计
- 跨大区独发**不允许**含同行表述（继承 03e §6.1 + 本文档 §6.4）

### 9.2 海外（FDA / EMA / 出海场景）

- 比较广告法在美国相对宽松（FTC 允许真实数据 + 公开来源比较广告）
- 但仍按本文档 §2 R1-R5 处理（统一最严标准；防止跨境 leak）
- 出海材料须海外法务 review（不在 v0.x.x 阶段重点）

### 9.3 港股投资者关系（HKEX 上市公司披露）

- 招股书 / 季报 / 年报中"竞争格局"段落是 HKEX 法定要求
- 必须按 HKEX Listing Rules + Inside Information Rules 处理（继承 03f §3.4 招股材料路径）
- 法务 + Sponsor + HKEX 联系人四方逐字签
- 不在投资者关系材料中含 vs 表 / 暗示

### 9.4 学术国际期刊（与海外 PI 合作）

- 按学术规范处理（§7）
- 不受本文档 §2 R1-R5 限制（学术发表不在适用范围 · §0.2）
- 但内部 review 流程仍走 03f §4.6 学术合作合同模板化

## 10. 模板库（10 个核心场景模板）

### 10.1 T-COMP-CUST-1 客户主动比较

**场景**：客户在演示中说"和 X 比"

```
"都是临床决策辅助产品，很多医院评估期会看多家。能问下你最关心
哪几个维度吗？比如 (1) 中文 (2) EHR 集成 (3) PITL 验证流程 (4) 数据合规。
基于你的需求，我们可以聚焦介绍 DE 的对应能力 + 真实数据。"
```

### 10.2 T-COMP-CUST-2 客户问优劣

**场景**：客户问"你们比 X 好在哪？"

```
"我们一般不直接评价同行。你能告诉我你为什么对比这两个产品吗？
是某个 [具体场景] 不达预期，还是 [集成 / 价格 / 合规] 维度有顾虑？
基于你说的，我们可以告诉你 DE 在 [具体场景] 的真实数据 [带 SourceID]。"
```

### 10.3 T-COMP-MEDIA-1 媒体询同行新功能

**场景**：记者问"X 公司发布了 [功能 Y]，你们怎么看？"

```
"感谢你的问题。我们不评价同行的具体产品。
如果说行业层面，临床 AI 产品的 [Y 功能 / Y 趋势]
反映了 [具体痛点] 的真实需求。我们 DE 在这一方向上的实践是
[具体描述 + 数据 + SourceID]。"
```

### 10.4 T-COMP-MEDIA-2 媒体探询监管

**场景**：记者问"X 公司被 NMPA 警告，你们怎么看？"

```
"我们看到了相关公告。我们一向严格按 [具体合规措施] 来设计 / 部署 DE，
比如 [PITL 全流程验证] 和 [数据本地化部署]。具体到 X 公司的情况，
我们没有内部信息也不会评价同行。"
```

### 10.5 T-COMP-RFP-1 RFP 要求 vs 表

**场景**：客户 RFP 文档要求填"与竞品对比表"

```
[投标响应文字]

"基于我方政策，我们不能在投标材料中直接对比同行产品。
然而，我们可以提供针对 RFP 中 [具体维度 1-12] 的能力描述 + 真实数据 + SourceID。

附件：
- 梅斯 AI · DeepEvidence 12 维度能力详述（基于公开数据 + 客户授权 case）
- 'AI 临床决策辅助产品评估框架 v1.2'（参考标准 · 不含品牌名）
- 5 家三甲医院落地 case study + 临床医生反馈引语 [按 03c §6.2 8 条护栏]"
```

### 10.6 T-COMP-RETRACT-1 同行论文被撤稿后客户询问

**场景**：客户问"X 公司基于的研究被撤稿对你们影响"

```
"我们了解到 [文献标题] 撤稿事件。我们的 DE 没有引用该研究 /
我们引用的是 [其他研究 + SourceID]。

补充：我们的实证流程 [03f §1 实证 L1-L4 + SourceID 注册体系]
是为了应对正是这种风险。每条结论引用的研究都注册并实时监测撤稿状态。"
```

### 10.7 T-COMP-INTERNAL-1 内部销售脚本（Battle Card 节选 · 不外发）

```
[Battle Card · 仅内部 · 不打印 · 不外发]

直接竞品 C1：[Watson Health]
公司：IBM 旗下；2022 年部分业务出售
核心定位：医疗 AI 解决方案 + 决策辅助
产品功能：基于其 2024 年公开白皮书 [SourceID]
公开数据：— [SourceID]
近期动态：— [SourceID]
NMPA 资质：— [SourceID]

DE 对应能力：
- [客观陈述 DE 的对应维度，不褒贬]
- [数据 + SourceID]

客户对话 talking points：
- 不点名 [Watson]
- 客户问比较时按 §4 4 步原则
- 聚焦"中文 / EHR 集成 / PITL 验证 / 临床本地化"
```

### 10.8 T-COMP-COMP-1 同行收购 / 退市后客户询问

**场景**：同行被收购后客户询"X 退市了，你们呢？"

```
"梅斯健康 2415.HK 是港交所主板上市公司，财务稳健 [基本面陈述]。
我们 AI 产品线的研发和服务都在持续投入。
如有具体合规 / 长期支持的顾虑，可以提供 [客户支持 SLA + 数据迁移政策 + 长期合同]。"
```

### 10.9 T-COMP-AVOID-1 同行客户主动接触

**场景**：同行某客户的人主动联系你说"我们想换供应商"

```
"很高兴你联系我。我们一向以产品价值取胜，不依赖挖角。
能告诉我你的具体需求吗？基于此我可以告诉你 DE 是否适合 [你的场景]。

注：不需要你提供 X 公司的任何内部信息 / 合同细节。"

→ 任何信息流入都不主动询问；如客户主动透露按 §10.10 处理
```

### 10.10 T-COMP-AVOID-2 客户主动提供同行内部信息

**场景**：客户在对话中主动提及同行内部信息（合同金额 / roadmap / 离职员工口述）

```
"感谢你的信任。但出于合规考虑，我们不会接收 X 公司的任何
保密信息 / 内部材料。请仅基于你的实际需求和我们的公开能力对话。"

→ 立即停止接收；不进入 Battle Card / 销售脚本 / 邮件留痕
→ 24h 内向法务报告（即使没问题，留 audit trail）
```

## 11. 14 条发布前自检清单

每个涉同行表述的物料发出前必须 ack 全部 14 条：

- [ ] 1. **R1 不点名**：物料中是否含同行品牌 / 产品名？（即使首字母）

- [ ] 9. **学术合作论文** 法务 + 临床顾问 + 论文 PI 三签？
- [ ] 10. **同行撤稿 / 监管事件** 公开沉默 + 仅内部 Battle Card 更新？
- [ ] 11. **Battle Card** 仅内部 + 不打印 + 不外发邮件附件？
- [ ] 12. **Battle Card 字段** 无 §3.3 严禁项（弱点 / 客户负评 / 内部消息 / 个人评价）？
