## 1. 过渡期定义

### 1.1 时间起止

| 起止 | 时间点 | 触发 |
|---|---|---|
| **起点** | 2026-05-05（v1.0 锁定）| HANDOFF §3 锁定决策 |
| **终点** | 母品牌 W2 摄影 + 插画库正式交付 | 母品牌团队公告 + 文档可下载 + 设计 / 市场 / 法务联合 ack |
| **估期** | 6–12 个月 | 基于 HANDOFF §10.4 母品牌团队后补项 timeline 预估 |

### 1.2 过渡期产出 = 临时素材

过渡期内产生的所有摄影 / 插画**视为临时素材**：

- 必须按 §4 临时素材库登记
- 必须季度复审（频次比 03f §2.4 客户 case 更严，因为过渡期合规风险更高）
- W2 交付后按 §6 迁移规程审计 + 决定保留或下架
- 不允许过渡期素材进入"永久品牌资产库"

### 1.3 过渡期前置约束（继承 04 §7 不变）

无论过渡期如何处理，下列红线**完全继承**：

- ❌ **AI 生成医学影像**（FTC 合规红线）
- ❌ **Stock photo 图库照片**（含 Getty / Shutterstock / 视觉中国 等）
- ❌ **emoji 装饰、贴纸、卡通人物**
- ❌ **患者面向材料中具象 AI 大脑 / 机器人 / 芯片图像**
- ❌ **AI 流程图人物剪影 / 医院建筑图**——必须 dot-spiral 几何抽象
- ❌ **真人医生代言形象**（继承 03a §6.2 / 全平台禁止）

## 2. 摄影：过渡期"安全可用"范围

### 2.1 可用源（按优先级）

| 优先级 | 来源 | 限制 | 入库要求 |
|---|---|---|---|
| **P0 首选** | 自摄 + 书面知情同意 | 拍摄前签同意书 + 法务 review 用途条款 | §2.4 SOP + 临时素材库 |
| **P1** | 母品牌已交付的现存照片 | 仅 04 §7 / `assets/master-brand/` 列出的 | 直接引用 母品牌资产路径 |
| **P2** | 创作共用 CC0 / Unsplash 商业可用 | **逐张法务审 + 来源链留痕** | 临时素材库（标注为"过渡期外采"，W2 后优先替换）|
| **P3** | 客户授权 case 照片 | 按 03f §1.2 L4 书面同意 + 03e §5.4 跨层规则 | 实证库 SourceID + 临时素材库交叉登记 |

**铁律**：

- **P0 优先**——所有**首次拍摄**默认走自摄 + 知情同意流程；只在自摄不可行时降级 P1 / P2
- **P2 不无脑用**——CC0 / Unsplash 看似免费，但商业用途授权链 / 模特肖像权 / 后续撤回风险都需逐张评估；任何 P2 素材必须法务签
- **P3 跨层不挪用**——T1 客户 case 照片不挪给 T4 物料（继承 03e §5.4）

### 2.2 内容白名单

继承 04 §7 + 扩展：

| 类型 | 适用 | 注意 |
|---|---|---|
| **真实临床场景** | 自摄 / 客户授权 | 不出现患者；医生角色按 03a §6.2 二分（代言禁 / 使用介绍允许）|
| **学术会议 / 论坛** | 自摄 / 公开授权 | 现场合影中医生作为多人之一无单独突出（继承 03a §6.2）|
| **办公环境 / 团队 / 团建** | 自摄 | 员工书面同意 |
| **活动签到 / 展位 / 海报** | 自摄 | 活动主办方授权 |
| **产品 demo 录屏 + 截屏** | 内部 | 不属于"摄影"严格范畴；按产品自己 demo 资产管理 |

### 2.3 过渡期摄影禁忌（**比 04 §7 + 母品牌 v1.1 §8 更严**）

- ❌ **模仿 stock photo 的情绪化摆拍**——如"医生握手"、"微笑摆拍听诊器"、"夕阳下背影"等情绪渲染
- ❌ **真人医生作为代言形象**（继承 03a §6.2）
- ❌ **真人医生独立特写 + 产品 logo / wordmark 邻近**（构成代言）
- ❌ **患者出镜**（即使匿名化处理，过渡期一律禁；W2 交付后视母品牌规范评估）
- ❌ **个人身份信息（PII）可见**——员工胸牌 / 工号 / 病例编号 / 患者床号必须模糊化或重拍
- ❌ **未经主体书面同意的人物摄影**（即使是"路人甲"——拍摄了就算）
- ❌ **后期过度处理**——磨皮 / 大幅滤镜 / 美颜 / 添加非现场元素
- ❌ **过渡期内不做"职业模特拍摄"**——成本高 + 与母品牌 W2 交付重复 + 易归 stock photo 风格
- ❌ **品牌色调外的照片**——后期色温保持 04 §7 自然光偏冷中性；不允许暖色 / 高饱和

### 2.4 过渡期摄影 SOP

```
[拍摄需求提出]（由市场 / 内容生产者）
       ↓
[填写《过渡期摄影申请表》]（用途 / 场景 / 对象 / 计划发布渠道）
       ↓
[市场负责人 + 法务联合签字]
       ↓
[准备拍摄器材 + 知情同意书模板]
       ↓
[拍摄现场]
       ├─ 拍摄前由对象签知情同意书（每位人物 1 份）
       ├─ 自然光优先；偏冷中性色温
       ├─ 不使用滤镜 / 美颜 app
       └─ 备份 RAW 文件
       ↓
[后期]
       ├─ 仅做色温微调 + 裁切 + 锐化
       ├─ 模糊 PII（胸牌 / 工号 / 病例编号 / 患者床号）
       └─ 导出 web / print 两版
       ↓
[入库]
       ├─ 文件命名：YYYYMMDD_场景_对象_序号.jpg
       ├─ 元数据：拍摄日期 / 摄影师 / 主体同意书 ID / 用途范围 / 有效期
       ├─ SourceID 注册（继承 03f §2.1）
       └─ 临时素材库标注（参 §4）
       ↓
[使用前对照 §2.3 禁忌自检 + §8 自检清单]
```

**关键模板**（v0.2 落地任务 · 法务交付）：

- `《过渡期摄影申请表》`
- `《过渡期摄影知情同意书》`（含转载范围 / 撤回机制 / 有效期 / 肖像权授权范围 / PII 模糊化承诺）

### 2.5 数据 / PII 模糊化处理

任何摄影中可见的 PII 必须**拍摄时回避**或**后期模糊化**：

- 员工胸牌：可见即模糊；如需展示身份用胸前别针 / 衣领领针替代
- 工号 / 工卡号：模糊
- 病例编号 / 床号 / 患者姓名 / 患者照片：禁止可见
- 屏幕显示的真实数据：模糊或替换为示意数据
- 名片 / 文件 / 白板：模糊或重拍

## 3. 插画：过渡期"安全可用"范围

### 3.1 dot-spiral 母题细化

继承 04 §4（CoT 几何）+ 母品牌 v1.1 §8.3 dot-spiral 母题。**过渡期使用规则**：

| 维度 | 锁定值 |
|---|---|
| **节点形状** | 圆点（不允许方点 / 三角 / 异形）|
| **节点尺寸** | 6px / 8px / 12px 三档（与 04 §4 CoT 6px 兼容；插画用 8px 或 12px 区分语境）|
| **节点间距** | 14–24px（按尺寸 × 2–3 倍）|
| **配色** | 单产品色 + 中性灰阶（neutral-300 / 500）；母品牌场景用 master brand `#0F62FE`；不混用 4 产品色 |
| **形态** | 直线 / 弧线（60° / 90°）/ 螺旋 / 网格——4 种基本形态，不混合 |
| **stroke** | 不超过 2px；节点之间用 stroke 连接时 stroke-color = 节点色 opacity 0.5 |
| **不允许** | 渐变 / 阴影 / 内发光 / 3D / 折射 / 故障艺术（glitch）|

### 3.2 可用源

| 优先级 | 来源 | 限制 |
|---|---|---|
| **P0 首选** | 自创几何抽象（dot-spiral 母题）| 设计师按 §3.1 规则手绘 / Figma 矢量 / SVG |
| **P1** | 母品牌 v1.1 已提供的抽象资产 | 仅 `assets/master-brand/` 列出的 |
| **P2** | AI 生成几何抽象 | **严格按 §3.3 限制**——仅几何 / 非医学 |
| **P3** | Open Source 几何资产 | 商业可用 license（CC0 / MIT / Apache）+ 法务签 + 来源留痕 |

### 3.3 AI 生成插画的限制（**FTC 红线扩展**）

继承 04 §7 "AI 生成医学影像绝对禁止"。**过渡期专属扩展**——AI 生成插画允许范围**仅限**：

- ✅ 几何抽象（点 / 线 / 几何形）
- ✅ 非医学语境
- ✅ 仅作为"灵感参考"——必须由设计师**重绘 / 转矢量**后入库；不允许 AI 直接生成稿成品入库

**绝对禁止 AI 生成**：

- ❌ 任何医学语境（器官 / 设备 / 影像 / 解剖）
- ❌ 人物（即使抽象）
- ❌ 患者形象
- ❌ 医生形象
- ❌ 医院建筑
- ❌ 药品 / 药丸 / 注射器 / 听诊器等具象医疗物品

**入库要求**：

- AI 生成稿不直接引用——必须**人工重绘 / 矢量化**后保存为新文件
- 保留 AI 生成稿原始 prompt 留痕（实证库登记，参 03f §2.1 来源元数据）
- 不允许产品 UI / 营销物料中"标注 AI 生成"——视为重绘后的设计师作品

### 3.4 过渡期插画禁忌（**比 04 §7 + 母品牌 v1.1 §8 更严**）

- ❌ **卡通 / Q 版 / 表情包风格**——无论"几何"还是"人物"
- ❌ **情绪化插画**——哭脸 / 怒脸 / 兴奋脸 / 受伤 robot 等情绪渲染（继承 04b §9.3 错误状态克制）
- ❌ **医学影像风格的"创作"**——X 光 / CT / 心电图 / 解剖图等具象医疗影像（即使是"艺术化"风格）
- ❌ **患者形象**——任何形式（剪影 / 抽象 / 卡通）
- ❌ **拟人化数据 / 拟人化 AI**——不允许"AI 角色"作为吉祥物 / mascot
- ❌ **故障艺术 / glitch / 像素艺术 / 涂鸦风格**——与品牌严肃性冲突
- ❌ **伪纹理**——不允许"水彩 / 油画 / 版画 / 像素"等模拟传统艺术风格
- ❌ **过度装饰**——一张插画中不超过 3 个层次（图层 / 元素 / 颜色）

### 3.5 应用场景

| 场景 | 推荐 |
|---|---|
| Hero 区装饰 | 自创 dot-spiral 几何（直线 / 弧线 / 螺旋 / 网格择一）|
| 流程示意图 | dot-spiral 节点 + stroke 连线（不用人物 / 设备具象）|
| 空状态占位（无照片）| 母品牌 placeholder box（参 §5）或自创 dot-spiral |
| 卡片背景 | dot-spiral 母题作为低饱和度水印（opacity ≤ 0.1）|
| 报告 / 白皮书 cover | 自创几何 + 产品色调 + 母品牌 endorsement |
| 社交头图 | 继承 03a §6.1 平台资产规范 |

## 4. 临时素材库

### 4.1 库结构

过渡期所有摄影 / 插画素材必须进入**临时素材库**——独立于实证库（避免污染主实证体系）：

**位置**：建议复用 03f §6.1 实证库工具（开源自建）的同一基础设施，作为"过渡期素材库"独立 schema/table。

**字段（强制）**：

- 素材 ID（建议格式 `INTERIM-YYYY-NNNN`，与 SourceID `EVID-YYYY-NNNN` 不混用）
- 类型（摄影 / 插画 / AI 重绘）
- 来源优先级（P0 / P1 / P2 / P3，参 §2.1 / §3.2）
- 来源元数据（自摄日期 + 摄影师 / CC0 链接 / 母品牌资产路径 / 客户合同号）
- 主体同意书 ID（如有人物）+ 同意书有效期 + 转载范围 / 撤回机制
- AI 生成 prompt（如适用，参 §3.3）
- 实证库 SourceID 关联（如对应 §1.2 客户案例）
- 法务审批状态（pending / approved / revoked）
- 录入日期 + 录入人 + 季度复审日期
- 用途记录（已发布物料反向追踪）
- W2 迁移状态（pending / migrated / replaced / archived，参 §6）

### 4.2 复审节奏

**比 03f §2.4 客户 case 更严**——过渡期素材风险更高（特别是 P2 CC0 / P3 第三方授权）：

| 来源 | 复审频次 |
|---|---|
| P0 自摄 | 季度（同意书有效期内）|
| P1 母品牌资产 | 半年（仅核对引用是否更新）|
| **P2 CC0 / Unsplash** | **季度**（来源链是否仍有效 / 授权是否撤回）|
| P3 客户授权 | 按客户合同期限或年度孰短（继承 03f §2.4） |
| AI 重绘稿 | 半年（核对设计师署名 / 重绘程度是否仍站得住）|

**过期未复审**：素材自动转 pending；任何依赖该素材的物料在新发布中不可用（与 03f §2.4 规则一致）。

### 4.3 用途反向追踪

每条素材必须记录**已使用的物料列表**——撤回素材 / W2 迁移时可一键查询影响范围（继承 03f §2.1 反向追踪）：

- 公众号文章 ID + 发布日期
- 白皮书 / PDF 名 + 版本号
- 网站页面 URL
- 社交媒体帖 ID
- 招股 / 投资者材料

### 4.4 素材撤回流程

主体撤回 / 法务发现违规 / 客户投诉 / 母品牌团队要求下架等触发：

```
[撤回触发]
       ↓
[临时素材库标记 revoked]
       ↓
[反向追踪所有依赖物料]
       ↓
[按 03f §3.4 物料分类处理]：
       ├─ 社交已发布 → 不删帖 + 加更新声明（继承 03a §5.1）
       ├─ 网站 / 落地页 → 立即更新 + archive 注明
       ├─ 白皮书 → 召回 PDF + 通知客户
       └─ 招股材料 → 按 HKEX 流程（继承 03f §3.4）
       ↓
[替换素材]：
       ├─ 自摄重拍（高优先）
       ├─ 母品牌 placeholder box（短期占位）
       └─ W2 交付资产（如已交付）
```

  border: 1px solid var(--neutral-200);

  justify-content: center;
  min-height: 120px;

  font-size: 12px;

**铁律**：

- placeholder box 是**最简单**的空状态方案——优先选择
- 12px 标签是上限（继承 04 §7）；不允许加图标 / 装饰 / 渐变
- aria-label 用 "占位图"或上下文相关词（如 "团队照片 占位"）

### 5.2 dot-spiral 抽象作为占位

如需更"有内容"的占位（如 hero 区），用 §3.1 dot-spiral 母题：

- 单产品色 + neutral 灰阶
- 占据视觉位但不传达具体信息
- 配文字（"团队照片 即将上线"等）

### 5.3 文字 + icon 占位

某些场景（错误页 / 空列表 / 空仪表盘）：

- 用 04 § Diff View 风格的文字占位
- 配 §3.1 dot-spiral 极简形态（如 5 个圆点静态排列）
- 不用情绪化插画（继承 §3.4 + 04b §9.3）

### 6.1 触发条件

母品牌团队公告 W2 交付 + 摄影 / 插画库文档可下载 + 设计 / 市场 / 法务联合 ack 资产可用——三者同时满足才视为"过渡期结束"。

### 6.2 迁移时间窗

**T+30 天内**完成现存临时素材审计：

| 阶段 | 动作 |
|---|---|
| T+0 | 公告过渡期结束；临时素材库进入"迁移期"状态 |
| T+7 | 完成现存临时素材清单 + 用途反向追踪（按 §4.3）+ 优先级排序（高曝光 → 低曝光）|

| T+30 | 低曝光素材替换或下架完成；本文档归档 |

### 6.3 替换 vs 保留分类

| 类型 | 处理 |
|---|---|
| **P0 自摄 + 优质** | 评估是否纳入 W2 永久库（设计 + 市场 + 法务联评）|
| **P0 自摄 + 一般** | 保留至 W2 资产覆盖该场景为止；逐步替换 |
| **P1 母品牌资产** | 自动并入 W2（升级元数据，不下架）|
| **P2 CC0 / Unsplash** | **优先替换**——W2 覆盖该场景后立即下架 P2 |
| **P3 客户授权 case** | 保留——客户授权独立于母品牌；按 03f §2.4 合同期限或年度孰短复审 |
| **AI 重绘稿** | 评估是否纳入 W2 永久库；如 W2 已有更优插画则替换 |

### 6.4 临时素材库归档

W2 交付后：

- 临时素材库**不删除**——保留作为合规审计 trail
- 状态转为"迁移完成 / 历史归档"
- 不再接受新增临时素材登记
- 已撤稿 / 替换素材保留 revoked 标记 + 反向追踪记录

### 6.5 本文档归档

W2 交付后本文档**归档**：

- 文件名加后缀：`04c-photography-illustration-interim.md` → `04c-photography-illustration-interim.archived.md`

- 04 §11 子文档清单状态从 v0.2 改为"已归档（W2 交付）"
- HANDOFF / discovery-report 同步标注

### 6.6 迁移期合规风险

迁移期内（T+0 至 T+30）的合规风险：

- 临时素材与 W2 资产**短期共存**——视觉一致性可能下降
- 未替换的 P2 / P3 素材仍合法（在合同 / CC0 license 内）；不构成违规
- 但**新启动**的物料**禁止**使用临时素材——必须用 W2 资产

**铁律**：迁移期内任何**新文案 / 新落地页 / 新白皮书**只能用 W2 资产；不允许"临时素材最后一搏"。
