React 包 · 9 段视觉验证截图存档

把 react/demo/index.html 的 9 段实时演示在 puppeteer headless chromium 下渲染并保存为 PNG，作为 v1.0 锁定时点的视觉决策档案。 未来 React 包改版前对照此档案，确保视觉不退化。 生成方式：node scripts/build-react-demo-screenshots.mjs。

⚠ 尚未生成截图，或截图已生成但通过 file:// 直接打开本页（被 CORS 阻止 fetch manifest）。

① 检查截图是否已生成：

ls preview/screenshots/react-demo/    # 应看到 9 个 PNG + manifest.json

② 若已生成，请通过 http 打开本页（任选一种）：

# A. VitePress 站点（推荐）
cd docs && npm run dev
# 打开 http://localhost:5173/preview/screenshots/index.html

# B. 临时静态服务器（kit 根）
python3 -m http.server 8080
# 打开 http://localhost:8080/preview/screenshots/index.html

# C. 通过 docs 站点的 demo 页（iframe 内嵌，自动 http）
# 打开 http://localhost:5173/demos/react-screenshots

③ 若尚未生成，在 kit 根目录运行：

cd react && npm install && npm run build       # 确保 dist 已构建
cd .. && npm install puppeteer                # 首次约 200MB chromium 下载
node scripts/build-react-demo-screenshots.mjs # 输出 9 张到 preview/screenshots/react-demo/

9 段

用法

当前用途：

  • 设计 / 工程 review 时：对照截图判断当前渲染是否还符合 v1.0 锁定的视觉决策
  • React 包升版前：先跑一次截图脚本生成新档案；与本档案 diff 看是否有意外退化
  • BD / 投资者材料：直接复用截图（注明 React 包版本）

什么时候重新跑：

  • React 包发新版（version bump）前后
  • 修改任何信任徽章 / Logo / Wordmark / 母品牌 logo 的 token 后
  • 每季度母品牌 v1.1 sync 后

不能用来替代的：

  • 静态截图不能验证 CoT 动画 / NanoCitation 交互 — 这些仍需 live demo 验证
  • 截图只是某一时刻的视觉，spec 才是规则的 source of truth

归档说明

每张截图同时记录在 preview/screenshots/react-demo/manifest.json，含：

  • 截图日期（ISO 8601）
  • React 包名+版本（如 @yogyoung-code/ai-brand-kit@0.1.0）
  • 9 段对应的 spec 锚点（点击跳转回锁定决策章节）
  • 每张 PNG 字节数（用于 diff 时快速看出哪段视觉变化）
